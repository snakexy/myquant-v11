## 4.1 V5场景化服务架构

### V5 vs V4 架构对比

| 对比项 | V4 统一管理器 | V5 场景化服务 |
|--------|--------------|--------------|
| **设计理念** | 一个服务管所有 | 每个服务专精一个场景 |
| **代码组织** | 3777行单文件 | 每个~500行模块化 |
| **数据源选择** | 统一路由逻辑 | 服务内部优化 |
| **扩展性** | 修改核心文件 | 添加新服务模块 |
| **测试维护** | 困难（互相影响） | 独立测试 |

### V5 架构图

```
┌─────────────────────────────────────────────────────┐
│                    应用层                            │
│               (FastAPI / 业务服务)                    │
└─────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                  V5 场景化服务层                      │
├──────────────┬──────────────┬──────────────┬─────────┤
│ 监控服务     │ 增量更新服务  │ K线服务      │ 转换服务 │
│Monitoring   │Incremental   │Kline        │Conversion│
├──────────────┼──────────────┼──────────────┼─────────┤
│热点发现      │填补缺失数据  │实时K线推送  │批量转换  │
│异常检测      │在线获取补充  │分钟聚合     │数据迁移  │
│实时行情      │智能缺失检测  │订阅缓存     │格式统一  │
└──────────────┴──────────────┴──────────────┴─────────┘
                          ↓
┌─────────────────────────────────────────────────────┐
│                    适配器层                           │
├──────────┬──────────┬──────────┬────────────────────┤
│ PyTdx2   │ XtQuant  │ TdxQuant │ LocalDB           │
└──────────┴──────────┴──────────┴────────────────────┘
```

### 四大场景服务定位

| 服务 | 主要场景 | 数据层级 | 优化策略 |
|------|----------|----------|----------|
| **MonitoringService** | 热点发现、异常检测 | L1 | 高速优先 |
| **IncrementalService** | 填补缺失数据 | L3 | 智能检测 |

## 4.2 监控服务配置 (MonitoringService)

### 服务定位

- **热点板块发现**
- **异常股票检测**
- **实时行情监控**

### 双层路由配置

```python
class MonitoringService:
    """
    实时监控服务 - V5场景化服务

    定位：热点发现、异常检测、实时行情
    优化：高速数据源优先
    """

    def __init__(self):
        # 第一层路由：板块 vs 非板块
        self.sector_router = {
            "enabled": True,  # 启用板块专用路由
            "primary": "TdxQuant",
            "reason": "586板块，6.99ms"
        }

        # 第二层路由：时间 + 数据类型
        self.data_router = {
            "trading_time": {
                "L1_snapshot": {
                    "primary": "TdxQuant",    # 0.60ms 最快
                    "secondary": "XtQuant",    # 6ms 缓存
                    "batch_threshold": 100     # >100股用XtQuant
                },
                "sector_data": {
                    "primary": "TdxQuant",    # 6.99ms, 586板块
                    "warning": "XtQuant不支持板块"
                }
            },
            "non_trading_time": {
                "L1_snapshot": {
                    "primary": "PyTdx2"       # 唯一可用
                },
                "sector_data": {
                    "primary": "PyTdx2",      # 唯一可用
                    "warning": "板块数量有限"
                }
            }
        }
```

### 板块数据获取示例

```python
def get_hot_sectors(self, limit=20):
    """
    获取热点板块 - 板块专用路由

    重要：
    - 必须使用板块专用路由（XtQuant不支持）
    - 交易时间：TdxQuant（6.99ms, 586板块）
    - 非交易时间：PyTdx2（~15ms，部分板块）
    """
    is_trading = self.is_trading_time()

    if is_trading:
        # 交易时间：TdxQuant绝对优势
        logger.info("使用 TdxQuant 获取板块数据（6.99ms, 586板块）")
        return self.tdxquant_adapter.get_sectors(limit)
    else:
        # 非交易时间：PyTdx2唯一可用
        logger.warning("非交易时间使用 PyTdx2（板块数量有限）")
        return self.pytdx2_adapter.get_sectors(limit)
```

### 快照数据获取示例

```python
def get_snapshot(self, symbols):
    """
    获取实时快照 - 非板块数据路由

    策略：
    - <100股：TdxQuant（0.60ms最快）
    - 100-300股：XtQuant（6ms缓存）
    - >300股：PyTdx2分批获取
    """
    is_trading = self.is_trading_time()
    count = len(symbols)

    if is_trading:
        if count <= 100:
            # 少量股票用TdxQuant（0.60ms最快）
            return self.tdxquant_adapter.get_snapshot(symbols)
        elif count <= 300:
            # 中等数量用XtQuant（6ms缓存）
            return self.xtquant_adapter.get_snapshot(symbols)
        else:
            # 大量股票用PyTdx2分批
            return self._batch_get_snapshot(symbols, batch_size=800)
    else:
        # 非交易时间只能用PyTdx2
        return self._batch_get_snapshot(symbols, batch_size=800)
```

## 4.3 增量更新服务配置 (IncrementalService)

### 服务定位

- **填补缺失的交易日数据**
- **在线获取最新数据**
- **数据完整性维护**

### 双层路由配置

```python
class IncrementalService:
    """
    增量更新服务 - V5场景化服务

    定位：填补缺失数据
    优化：智能缺失检测 + 增量获取
    """

    def __init__(self):
        # 第一层路由：板块 vs 非板块
        self.sector_router = {
            "enabled": True,
            "primary": "TdxQuant",  # 交易时间
            "fallback": "PyTdx2"     # 非交易时间
        }

        # 第二层路由：当天分钟线 vs 历史K线
        self.data_router = {
            "intraday_kline": {
                "trading_time": {
                    "primary": "XtQuant",    # 0.90ms
                    "secondary": "PyTdx2",   # 10-19ms
                    "excluded": "TdxQuant"   # ❌ 不支持
                },
                "non_trading_time": {
                    "primary": "PyTdx2"      # 唯一可用
                }
            },
            "history_kline": {
                "any_time": {
                    "primary": "QLib",       # 7-10ms 本地
                    "secondary": "PyTdx2",   # ~19ms
                    "tertiary": "TdxQuant"   # ~50ms (仅交易时间)
                }
            }
        }
```

### 当天分钟线填补示例

```python
def fill_intraday_gaps(self, symbols, dates):
    """
    填补当天分钟线缺失

    关键限制：
    - ❌ TdxQuant不支持当天分钟线
    - ✅ XtQuant 交易时间最优（0.90ms）
    - ✅ PyTdx2 非交易时间唯一

    策略：
    交易时间：XtQuant → PyTdx2
    非交易时间：PyTdx2
    """
    is_trading = self.is_trading_time()

    if is_trading:
        # 交易时间：优先XtQuant（TdxQuant不支持）
        try:
            logger.info("使用 XtQuant 获取当天分钟线（0.90ms）")
            return self.xtquant_adapter.get_minute_kline(symbols, dates)
        except Exception as e:
            logger.warning(f"XtQuant失败，降级到PyTdx2: {e}")
            return self.pytdx2_adapter.get_kline(symbols, period='1min')
    else:
        # 非交易时间：只能用PyTdx2
        logger.info("非交易时间使用 PyTdx2 获取当天分钟线（10-19ms）")
        return self.pytdx2_adapter.get_kline(symbols, period='1min')
```

## 4.4 K线服务配置 (KlineService)

### 服务定位

- **实时K线订阅推送**
- **分钟级K线聚合**
- **实时K线图表数据**

### 双层路由配置

```python
class KlineService:
    """
    实时K线服务 - V5场景化服务

    定位：实时K线推送、分钟聚合
    优化：订阅+缓存组合
    """

    def __init__(self):
        # L0 订阅配置
        self.subscription_config = {
            "XtQuant": {
                "capacity": 300,          # 300股上限
                "latency": "<1ms",        # 推送延迟
                "priority": 1             # 优先
            },
            "TdxQuant": {
                "capacity": 100,          # 100股上限
                "latency": "5-6ms",       # 轮询间隔
                "priority": 2
            }
        }

        # 当天分钟线获取
        self.intraday_kline_router = {
            "trading_time": {
                "primary": "XtQuant",     # 0.90ms
                "secondary": "PyTdx2",    # 10-19ms
                "excluded": "TdxQuant"    # ❌ 不支持
            },
            "non_trading_time": {
                "primary": "PyTdx2"       # 唯一可用
            }
        }

        # 历史K线
        self.history_kline_router = {
            "any_time": {
                "primary": "QLib",        # 7-10ms
                "secondary": "PyTdx2"      # ~19ms
            }
        }
```

### 实时分钟线推送示例

```python
def get_realtime_minute_kline(self, symbol):
    """
    获取实时分钟K线 - 每分钟更新

    策略：
    - 交易时间：XtQuant（0.90ms）→ PyTdx2
    - 非交易时间：PyTdx2（唯一可用）
    - 缓存：Redis TTL 60秒

    关键限制：❌ TdxQuant不支持当天分钟线
    """
    is_trading = self.is_trading_time()

    # 检查Redis缓存
    cache_key = f"kline:1min:{symbol}"
    cached = self.redis.get(cache_key)
    if cached:
        return json.loads(cached)

    # 获取最新数据
    if is_trading:
        # 交易时间：优先XtQuant
        try:
            logger.info(f"使用 XtQuant 获取 {symbol} 当天分钟线（0.90ms）")
            kline = self.xtquant_adapter.get_market_kline(symbol, period='1min')
        except Exception as e:
            logger.warning(f"XtQuant失败，使用PyTdx2: {e}")
            kline = self.pytdx2_adapter.get_kline(symbol, period='1min', count=240)
    else:
        # 非交易时间：只能用PyTdx2
        logger.info(f"非交易时间使用 PyTdx2 获取 {symbol} 当天分钟线（10-19ms）")
        kline = self.pytdx2_adapter.get_kline(symbol, period='1min', count=240)

    # 缓存60秒
    self.redis.setex(cache_key, 60, json.dumps(kline))
    return kline
```

### 订阅推送示例

```python
def start_subscription(self, symbols):
    """
    启动订阅推送 - L0层

    策略：
    - <300股：XtQuant订阅（<1ms推送）
    - <100股：可考虑TdxQuant订阅（5-6ms轮询）

    注意：
    - TdxQuant：100股上限，轮询模式
    - XtQuant：300股上限，事件推送
    """
    count = len(symbols)

    if count <= 300:
        # 优先用XtQuant
        logger.info(f"启动 XtQuant 订阅（{count}股）")
        self.xtquant_adapter.subscribe(
            symbols,
            callback=self._on_quote_update
        )
    elif count <= 400:
        # 超过300，需要双订阅
        logger.info(f"启动双订阅：XtQuant(300) + TdxQuant(100)")
        self.xtquant_adapter.subscribe(
            symbols[:300],
            callback=self._on_quote_update
        )
        self.tdxquant_adapter.subscribe(
            symbols[300:400],
            callback=self._on_quote_update_tdx
        )
    else:
        logger.warning(f"股票数量过多({count})，考虑分批处理")
```

```python
class ConversionService:
    """
    数据转换服务 - V5场景化服务

    定位：批量转换、历史迁移
    优化：直接读文件，不使用路由
    """

    def __init__(self):
        # 不使用在线数据源，直接读本地文件
        self.source_type = "TdxLocal"  # 通达信本地.day文件
        self.target_type = "QLib"      # Qlib格式

        # 不需要路由配置
        self.routing_enabled = False
```

### 批量转换示例

```python
def convert_batch(self, symbols, start_date, end_date):
    """
    批量转换数据

    策略：
    - 直接读取本地.day文件
    - 不使用在线API
    - 批量处理优化
    """
    results = {}

    for symbol in symbols:
        try:
            # 直接读本地文件
            day_file = f"E:/new_tdx64/vipdoc/{symbol}.day"
            data = self._read_day_file(day_file)

            # 转换为Qlib格式
            qlib_data = self._convert_to_qlib_format(data)

            # 保存到QLib数据库
            self._save_to_qlib(symbol, qlib_data)

            results[symbol] = {"status": "success", "records": len(qlib_data)}

        except Exception as e:
            logger.error(f"转换 {symbol} 失败: {e}")
            results[symbol] = {"status": "failed", "error": str(e)}

    return results
```

## 配置总结

### V5四大场景服务配置速查

| 服务 | 主要场景 | 板块数据路由 | 当天分钟线路由 | 快照路由 |
|------|----------|-------------|---------------|----------|
| **MonitoringService** | 热点发现 | TdxQuant(交易) → PyTdx2 | - | TdxQuant → XtQuant |
| **IncrementalService** | 填补缺失 | TdxQuant(交易) → PyTdx2 | XtQuant(交易) → PyTdx2 | - |
| **KlineService** | K线推送 | TdxQuant(交易) → PyTdx2 | XtQuant(交易) → PyTdx2 | 订阅优先 |
| **ConversionService** | 批量转换 | 直接读本地文件 | 直接读本地文件 | 不使用 |

### 关键配置要点

1. **板块数据必须用专用路由**
   - XtQuant完全不支持
   - TdxQuant交易时间最强
   - PyTdx2非交易时间唯一

2. **当天分钟线TdxQuant不支持**
   - 交易时间：XtQuant最优
   - 非交易时间：PyTdx2唯一

3. **快照数据按数量选择**
   - <100股：TdxQuant（0.60ms）
   - 100-300股：XtQuant（6ms）
   - >300股：PyTdx2分批

4. **历史K线本地库优先**
   - QLib（7-10ms）本地库
   - PyTdx2在线补充

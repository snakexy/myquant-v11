# XtQuant 订阅策略设计

> 研究日期: 2026-02-05
> 目的: 设计大型量化系统的智能订阅策略

## 订阅需求分析

### 1. 按范围分类

| 范围类型 | 股票数量 | 使用场景 | 订阅策略 |
|---------|---------|---------|---------|
| **全局订阅** | ~5000只（全A股） | 大盘监控、指数计算 | ⚠️ **需要分片** |
| **热门股订阅** | ~100-500只 | 热门板块、龙头股 | ✅ **单实例或多实例** |
| **自选股订阅** | ~10-100只 | 用户自选股 | ✅ **单实例足够** |
| **单股订阅** | 1只 | 详情页、实时分析 | ✅ **按需订阅** |

### 2. 按场景分类

| 场景 | 数据需求 | 实时性要求 | 订阅建议 |
|------|---------|-----------|---------|
| **行情列表页** | 简要行情（价格、涨跌幅） | 高（3秒） | ✅ **订阅** |
| **股票详情页** | 完整K线、分笔、财务 | 中 | ✅ **订阅 + 按需** |
| **K线图表** | 历史K线 + 实时更新 | 中（盘中5秒） | ✅ **订阅最新** |
| **技术分析** | 多周期K线 | 低 | ❌ **不订阅，按需获取** |
| **回测系统** | 历史数据 | 低 | ❌ **不订阅** |

### 3. 按模式分类

#### 订阅模式（订阅推送）
- ✅ **优点**: 实时推送、延迟低、服务器压力小
- ❌ **缺点**: 需要维护订阅列表、有上限
- 🎯 **适用**: 高频访问、实时性要求高的场景

#### 非订阅模式（按需获取）
- ✅ **优点**: 无上限、灵活
- ❌ **缺点**: 每次都请求、延迟高、服务器压力大
- 🎯 **适用**: 低频访问、历史数据查询

## 订阅策略设计

### 策略1: 全局订阅（分片）⭐推荐

**目标**: 监控全A股（~5000只）

**方案A: 单实例分批轮询**
```python
# 将5000只股票分为17批（每批300只）
batches = split_stocks(all_stocks, batch_size=300)

for batch in batches:
    # 订阅当前批次
    for stock in batch:
        subscribe_quote(stock, period='1d')

    # 获取数据
    quotes = get_full_tick(batch)

    # 处理数据
    process(quotes)

    # 取消订阅
    for stock in batch:
        unsubscribe_quote(stock)

    # 等待几秒再处理下一批
    time.sleep(1)
```

**方案B: 多实例并发** ✅ **最优**
```python
# 假设有17个MiniQMT实例
# 每个实例订阅300只

instances = [Instance(i) for i in range(17)]

# 分配股票
batches = split_stocks(all_stocks, batch_size=300)

for instance, batch in zip(instances, batches):
    instance.subscribe(batch)
    instance.start_monitoring()

# 并发获取所有数据
while True:
    all_quotes = {}
    for instance in instances:
        quotes = instance.get_cached_quotes()
        all_quotes.update(quotes)

    process(all_quotes)
    time.sleep(1)
```

**性能对比**:
| 方案 | 延迟 | 吞吐量 | 复杂度 |
|------|------|--------|--------|
| 单实例轮询 | ~17秒 | 5000只/17秒 | 低 |
| 多实例并发 | ~1秒 | 5000只/秒 | 中 |

### 策略4: 智能订阅（混合）⭐推荐

**目标**: 根据访问频率自动决定是否订阅

**算法**:
```python
class SmartSubscriptionManager:
    def __init__(self, limit=300):
        self.limit = limit
        self.subscribed = set()
        self.access_count = {}  # 股票 -> 访问次数
        self.last_access = {}   # 股票 -> 最后访问时间

    def access(self, stock):
        # 记录访问
        self.access_count[stock] = self.access_count.get(stock, 0) + 1
        self.last_access[stock] = time.time()

        # 检查是否需要订阅
        if stock not in self.subscribed:
            if len(self.subscribed) < self.limit:
                # 有空位，直接订阅
                self.subscribe(stock)
            else:
                # 无空位，检查是否替换
                self.maybe_replace(stock)

    def maybe_replace(self, new_stock):
        # 找出访问次数最少的已订阅股票
        min_stock = min(
            self.subscribed,
            key=lambda s: self.access_count.get(s, 0)
        )

        # 如果新股票访问更多，替换
        if self.access_count.get(new_stock, 0) > self.access_count.get(min_stock, 0):
            self.unsubscribe(min_stock)
            self.subscribe(new_stock)

    def subscribe(self, stock):
        subscribe_quote(stock, period='1d')
        self.subscribed.add(stock)
        logger.info(f"订阅: {stock} (已订阅: {len(self.subscribed)})")

    def unsubscribe(self, stock):
        unsubscribe_quote(stock)
        self.subscribed.discard(stock)
        logger.info(f"取消订阅: {stock} (已订阅: {len(self.subscribed)})")
```

**使用**:
```python
manager = SmartSubscriptionManager(limit=300)

# 每次访问股票数据时
def get_stock_quote(stock):
    # 记录访问
    manager.access(stock)

    # 获取数据（优先从订阅缓存）
    if stock in manager.subscribed:
        return get_full_tick([stock])[stock]
    else:
        # 非订阅，在线获取
        return get_market_data([stock])
```

# 显示K线图
def show_kline_chart(stock, period='1d'):
    # 获取历史K线（不依赖订阅）
    history = get_kline(stock, period=period, count=100)

    # 订阅最新数据（只在交易时间）
    if is_trading_time():
        subscribe_quote(stock, period=period)

        # 启动后台更新线程
        def update_thread():
            while is_trading_time():
                latest = get_full_kline([stock], period=period, count=1)
                update_chart(latest)
                time.sleep(5)

        Thread(target=update_thread).start()

    return history

## 订阅管理架构

### 1. 分层管理

```
全局订阅层（5000只）
    ↓
热门订阅层（500只）
    ↓
自选订阅层（100只）
    ↓
单股订阅层（1只）
```

### 2. 优先级策略

| 优先级 | 范围 | 订阅保证 |
|--------|------|---------|
| P0 | 自选股 | 100% 订阅 |
| P1 | 热门股 | 80% 订阅（智能轮询） |
| P2 | 全局 | 20% 订阅（分片轮询） |

### 3. 自动降级

```python
def get_quote(stock):
    # 优先级1: 从订阅缓存获取
    if stock in subscription_cache:
        return subscription_cache[stock]

    # 优先级2: 尝试订阅后获取
    if can_subscribe():
        subscribe(stock)
        return get_cached_quote(stock)

    # 优先级3: 在线获取
    return get_online_quote(stock)
```

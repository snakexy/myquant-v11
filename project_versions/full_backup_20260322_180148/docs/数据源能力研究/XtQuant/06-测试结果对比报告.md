# XtQuant 测试结果对比报告

> 测试日期: 2026-02-05
> 目的: 找到适配器失败的根本原因

## 问题回顾

**现象**: 测试脚本可以获取数据，但适配器获取失败

## 对比分析

### 差异1: start_time/end_time 参数

| 方式 | 测试结果 | 说明 |
|------|---------|------|
| 不传参数 | ✅ 成功（16条） | **推荐** |
| 空字符串 '' | ✅ 成功（16条） | 可用 |
| 传 0 | ✅ 成功（16条） | 可用 |

**结论**: 这**不是问题根源**

---

### 差异2: dividend_type 参数 ⭐关键

| 测试 | dividend_type值 | 结果 |
|------|----------------|------|
| 测试2 | 'none' (字符串) | ✅ 成功 |
| 测试3 | 0 (数字) | ❌ **报错** |
| 适配器(单个) | 'none' (字符串) | ✅ 应该成功 |
| 适配器(批量) | 0 (数字) | ❌ **失败！** |

**根本原因**:
```python
# ✅ 单个获取 (654行) - 正确
dividend_type_map = {"none": 'none', "front": 'front', "back": 'back'}
dividend_type = dividend_type_map.get(adjust_type, 'front')

# ❌ 批量获取 (842行, 918行) - 错误！
dividend_type_map = {"none": 0, "front": 1, "back": 2}
dividend_type = dividend_type_map.get(adjust_type, 1)
```

**这就是批量获取失败的唯一原因！**

---

## 修复方案

### 修复位置1: xtquant_dual_instance_adapter.py:842

```python
# ❌ 修改前
dividend_type_map = {"none": 0, "front": 1, "back": 2}
dividend_type = dividend_type_map.get(adjust_type, 1)

# ✅ 修改后
dividend_type_map = {"none": 'none', "front": 'front', "back": 'back'}
dividend_type = dividend_type_map.get(adjust_type, 'front')
```

## 关键教训

**一个参数类型错误导致整个批量功能不可用！**

这验证了您的观点：
> "前期必须彻底研究，否则后期各种问题太麻烦"

### 📊 性能数据

#### 实时快照性能
- **TdxQuant**: 0.60ms（最快）🥇
- **XtQuant**: 0.90ms（第二快）🥈
- **PyTdx**: 14ms（较慢）🥉

#### 批量获取性能
- **XtQuant**: 0.20ms/股（批量最优）🥇
- **TdxQuant**: 1.61ms/股（批量良好）🥈
- **PyTdx**: 2.79ms/股（批量中等）🥉

#### K线数据量
- **TdxQuant**: 1088根（最多）🥇
- **PyTdx**: 500根（中等）🥈
- **XtQuant**: 0根（不可用）🥉

### 🎯 核心优势

#### 1. L1实时快照最快
- **延迟**: 0.60ms
- **对比**: 比XtQuant快1.5倍，比PyTdx快23.6倍
- **适用场景**: 高频快照查询

#### 2. 板块数据优势
- **板块指数**: 完整支持
- **成分股列表**: 完整支持
- **板块实时行情**: 0.60ms最快
- **地位**: 🏆 **板块数据之王**

#### 3. 盘中辅助信息丰富
- **资金流向**: 实时资金流向数据
- **买卖盘口**: 五档盘口数据
- **实时分笔**: 逐笔成交数据

#### 4. 24/7可用
- **可用时间**: 全天候可用
- **限制**: 不受交易时间限制

### ⚠️ 主要限制

#### 1. 实时数据限制
- ⚠️ **仅快照数据**: 不支持实时推送
- ⚠️ **订阅功能未验证**: 虽然知道可以订阅，但推送机制不清楚
- ⚠️ **不是完整序列**: 快照只是当前时间点的数据

#### 2. 历史数据限制
- ⚠️ **不支持当天分钟数据**: 只能获取历史分钟数据（昨天及以前的，历史分钟数据**是支持的**）
- ⚠️ **K线数据量受限**: 最多1088根日线/周线/月线/1分钟/5分钟

#### 3. 其他限制
- ⚠️ **订阅上限**: 300只/实例（但功能未验证）
- ⚠️ **数据格式**: 返回格式可能需要转换处理

## 使用建议

### 最佳使用场景 ✅

1. **板块数据查询**（L5）
   - 板块指数数据
   - 成分股列表
   - **这是TdxQuant的主要优势**

2. **实时快照获取**（L1）
   - 0.60ms超快速度
   - 适合高频快照查询

3. **历史K线补充**（L2）
   - 短期历史K线（1000根以内）
   - 历史快照数据

4. **财务数据查询**（L4）
   - GP1-GP46完整财务指标
   - 财务分析

### 不适用场景 ❌

1. **实时推送订阅**（L0）
   - 不支持或未验证
   - 应使用XtQuant订阅

2. **当天分钟数据**（L2）
   - 不支持当天（今天）的分钟数据（历史分钟数据是支持的）
   - 应使用QLib本地DB或XtQuant

3. **超长K线数据**（L3）
   - 最多1088根
   - 应使用QLib本地DB

## 总结

**TdxQuant在MyQuant系统中的定位**:

| 数据类型 | TdxQuant支持 | 优先级 | 说明 |
|---------|-------------|--------|------|
| REALTIME_QUOTE | ✅ L1快照0.60ms | 优先级1 | **最快** |
| KLINE | ⚠️ 最多1088根 | 优先级2 | 不支持当天 |
| FINANCIAL | ✅ GP1-GP46 | 优先级1 | 完整支持 |
| SECTOR_INDEX | ✅ 板块数据 | 优先级1 | **板块数据之王** |
| SECTOR_COMPONENTS | ✅ 成分股 | 优先级1 | **板块数据之王** |
| L0订阅 | ❌ 未验证 | - | 不推荐 |

**核心建议**:
1. **板块数据**: TdxQuant是**唯一选择**（主要优势）
2. **实时快照**: TdxQuant是**最优选择**（0.60ms最快）
3. **历史数据**: 应使用QLib本地DB（数据量无限制）
4. **实时推送**: 应使用XtQuant订阅（已验证）

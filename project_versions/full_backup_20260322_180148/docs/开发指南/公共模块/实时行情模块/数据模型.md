## 🎯 模块定位

实时行情模块的数据模型定义了前后端交互的核心数据结构：**K 线数据**和**行情快照**。

数据流方向：
```
数据源(原始格式) → V5适配器(归一化) → Service层(业务处理) → API层(JSON) → 前端(渲染)
```

## 📊 数据结构

### 1. KlineData — 单根K线

**用途**: K 线图表绘制、技术分析

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| time | int | 是 | Unix 时间戳（毫秒，UTC） |
| open | float | 是 | 开盘价 |
| high | float | 是 | 最高价 |
| low | float | 是 | 最低价 |
| close | float | 是 | 收盘价 |
| volume | float | 是 | 成交量（股） |
| amount | float | 否 | 成交额（元） |
| color | string | 否 | 涨跌颜色：`#ef5350` 红涨 / `#26a69a` 绿跌 |

**Python 定义** (`backend/api/v1/quotes/kline.py:28`):
```python
class KlineData(BaseModel):
    time: int = Field(..., description="Unix时间戳（毫秒）")
    open: float
    high: float
    low: float
    close: float
    volume: float
    amount: Optional[float] = None
    color: Optional[str] = None  # #ef5350 红涨 #26a69a 绿跌
```

**TypeScript 定义**:
```typescript
interface KlineData {
  time: number       // Unix时间戳（秒），lightweight-charts 要求秒
  open: number
  high: number
  low: number
  close: number
  volume: number
  amount?: number
  color?: string
}
```

**转换规则**:
- 后端返回毫秒时间戳 → 前端除以 1000 转为秒（lightweight-charts 要求）
- 后端返回北京时间（naive datetime）→ 转为 UTC 时间戳

| symbol | string | 是 | 股票代码 |

### 5. TimePhase — 市场时段枚举

**用途**: 判断当前交易状态，驱动数据源选择和刷新策略

**代码位置**: `backend/data/utils/trading_time.py`

| 枚举值 | 时间范围 | 数据源策略 | 刷新间隔 | 缓存 TTL |
|--------|---------|-----------|---------|---------|
| `PRE_MARKET` | 0:00-9:15 | PyTdx | 30s | 86400s |
| `CALL_AUCTION` | 9:15-9:25 | TdxQuant → XtQuant | 3s | 10s |
| `WAIT_FOR_OPEN` | 9:25-9:30 | TdxQuant → XtQuant | 3s | 10s |
| `MORNING_OPENING` | 9:30-10:00 | TdxQuant → XtQuant | 3s | 300s |
| `MORNING_ACTIVE` | 10:00-11:30 | TdxQuant → XtQuant | 5s | 300s |
| `LUNCH_BREAK` | 11:30-13:00 | PyTdx | 30s | 86400s |
| `AFTERNOON_OPENING` | 13:00-14:00 | TdxQuant → XtQuant | 3s | 300s |
| `AFTERNOON_STABLE` | 14:00-15:00 | TdxQuant → XtQuant | 5s | 300s |
| `AFTER_MARKET` | 15:00-24:00 | PyTdx | 30s | 86400s |
| `WEEKEND` | 全天 | PyTdx | 30s | 86400s |

**关键方法**:

```python
TradingTimeChecker.is_trading_time()    # bool: 9:15-15:00 工作日
TradingTimeChecker.can_refresh_kline()  # bool: 同上
```

## 🔗 数据转换规则

### 时间戳处理

```
数据源 → 适配器 → API → 前端

TdxQuant/XtQuant/PyTdx 返回:
  naive datetime (北京时间, UTC+8)
      │
      ▼
适配器层: pd.DataFrame['datetime'] 列
      │
      ▼
API 层: _to_unix_timestamp()
  - naive datetime → 添加 UTC+8 时区 → 转换为 UTC → 毫秒时间戳
      │
      ▼
前端: lightweight-charts
  - 毫秒 ÷ 1000 → 秒时间戳
  - 图表显示本地时间
```

**关键**: 数据源返回的是**北京时间无时区 datetime**，API 层负责转换为 **UTC 毫秒时间戳**。

### 字段名映射

不同数据源返回的字段名不统一，适配器层负责归一化：

| 标准字段 | TdxQuant | XtQuant | PyTdx |
|---------|----------|---------|-------|
| price | current | lastPrice | price |
| change | price - last_close | - | price - last_close |
| change_percent | (change/last_close)*100 | - | (change/last_close)*100 |
| name | name | - | name |
| volume | vol | volume | vol |
| amount | amount | amount | amount |

适配器层的 `V5DataAdapter._normalize_quote_dict()` 方法完成此映射。

### 日K线空白填充

仅对日线（1d）生效：

```python
def _fill_missing_dates(df):

    周末/节假日 → 使用前一日收盘价填充 OHLC
    成交量、成交额 → 0
    """
    # 输入: 交易日数据 (可能跳过周末)
    # 输出: 连续日历日数据 (含周末，OHLC 持平)
```

**目的**: lightweight-charts 的日 K 线需要连续时间轴，不填充会导致周末空白区域。

## 📝 数据操作示例

### 后端获取K线

```python
from data_providers.adapters.v5.pytdx_adapter import V5PyTdxAdapter

adapter = V5PyTdxAdapter()
klines = adapter.get_kline(symbols=["600000.SH"], period="1d", count=100)

# klines 是 {symbol: DataFrame} 字典
df = klines["600000.SH"]
# df 列: datetime, open, high, low, close, volume, amount
```

### 后端获取快照

```python
from data_providers.adapters.v5.tdxquant_adapter import V5TdxQuantAdapter

adapter = V5TdxQuantAdapter()
quotes = adapter.get_quote(["600000.SH", "000001.SZ"])

# quotes 是 {symbol: dict} 字典
quote = quotes["600000.SH"]
# dict 键: price, change, name, bid1, ask1, ...
```

### 前端调用K线

```typescript
// 正确方式：通过 api/modules/ 调用（待实现）

  time: k.time / 1000,  // 毫秒 → 秒
```

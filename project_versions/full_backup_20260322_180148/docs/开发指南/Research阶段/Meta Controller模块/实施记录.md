## 🔧 实现细节

### Service层: `meta_controller_service.py`

**核心类**:
- `MetaControllerService` - 主服务类

**核心方法**:
- `optimize_factor_combination()` - 因子组合优化
  - 支持多种优化方法: ic_weighted, equal_weight, ir_weighted, auto
  - 自动计算最优权重配置
  - 保存优化结果为.pkl文件

- `select_best_model()` - 模型选择
  - 支持模型: lightgbm, xgboost, rf, mlp
  - 自动训练和评估多个模型
  - 返回最佳模型配置

- `optimize_hyperparameters()` - 超参数优化
  - 支持搜索方法: grid（网格搜索）, random（随机搜索）
  - 基于sklearn的GridSearchCV和RandomizedSearchCV
  - 交叉验证评估

**数据管理**:
- 优化结果保存为.pkl格式（符合项目架构）
- 存储路径: `backend/data/meta_controller/results/`
- 完整的优化历史管理

### API层: `meta_router.py`

**端点实现**:
1. `POST /api/v1/research/meta/factor/optimize` - 因子组合优化
2. `POST /api/v1/research/meta/model/select` - 模型选择
3. `POST /api/v1/research/meta/hpo/auto` - 超参数优化
4. `GET /api/v1/research/meta/trials` - 优化历史
5. `GET /api/v1/research/meta/result/{id}` - 结果详情
6. `GET /api/v1/research/meta/statistics` - 统计信息
7. `GET /api/v1/research/meta/health` - 健康检查

**请求/响应格式**:
- 统一使用Pydantic数据模型验证请求
- 统一的响应格式: `{code, message, data, timestamp}`
- 完整的错误处理

# Validator - 代码验证器

修改后的自动验证，确保代码质量和数据正确性。

## 触发条件

- 完成代码修改（Write/Edit）
- 需要验证修改是否正确
- 数据量可能出现异常时

## 验证清单

### 1. 语法检查
- Python：缩进、语法错误、未定义变量
- TypeScript/Vue：类型错误、语法错误

### 2. 逻辑检查
- 变量使用前已定义
- 函数参数匹配
- 返回值处理正确

### 3. 数据量检查（关键！）
- 检查是否有明显的数据量异常
- 如：从 5892 条变成 4 条 → 严重错误

### 4. 架构一致性
- 新代码遵循现有模式
- 无重复代码

### 5. 边界情况
- 空值处理
- 异常分支
- 资源释放

## 使用方式

```
/validate <文件路径>
```

或

```
Agent: "验证 backend/src/myquant/api/dataget/kline.py 的修改：\n1. 语法是否正确\n2. 逻辑是否有漏洞\n3. 数据量是否正常\n4. 是否符合项目规范"
```

## 严重错误判定

以下情况必须回退：
1. 数据量异常变化（>50% 变化）
2. 核心功能被删除
3. 明显的逻辑错误
4. 违反架构规范

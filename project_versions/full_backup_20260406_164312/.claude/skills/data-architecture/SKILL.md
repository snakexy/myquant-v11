# MyQuant V5 数据架构规范

## ⚠️ 触发条件

**涉及以下修改时，必须先读本 skill:**
- 修改 Service 层（KlineService、SeamlessKlineService、HotDBService 等）
- 修改 Adapter 层（任何 *_adapter.py 文件）
- 修改数据流向（API → Service → Adapter → 数据源）
- 新增数据源或修改数据源选择逻辑

**多级缓存策略**：

| 层级 | 名称 | 技术 | 延迟 | 容量 | 适用场景 |
|------|------|------|------|------|----------|
| 前端 L0 | IndexedDB | MyQuantKlineDB | <10ms | 100只×5周期 | 页面刷新恢复、增量更新 |
| 后端 L0 | mmap | MmapKlineStore | <0.1ms | ~100只 | 自选股热数据 |
| 后端 L1 | 内存 | InMemoryKlineStore | <1ms | ~100只 | 自选股预加载 |
| 后端 L2 | Redis | 跨进程缓存 | 1-5ms | 大 | 复权因子、会话 |
| 后端 L3 | 文件 | HotDB/LocalDB | 10-50ms | 无限 | 持久化存储 |

**访问优先级**：
- **前端**：`IndexedDB → HTTP API → WebSocket`
- **后端**：`L0 mmap → L1 内存 → L2 Redis → L3 文件 → 在线源`

**缓存层设计说明**：
- **前端 L0 IndexedDB** (v5.2 新增)：页面刷新后从本地恢复，减少 99% 数据传输，LRU + 7天过期
- **后端 L0 mmap**：内存映射文件，进程重启后数据仍在，零拷贝访问
- **后端 L1 内存**：进程内单例，紧凑存储 40B/记录，适合高频访问
- **后端 L2 Redis**：跨进程共享（FastAPI、风控、量化交易等多个 worker）
- **后端 L3 文件**：HotDB 是硬盘存储，需要 L2 缓存加速；FACTOR 表计算成本高，缓存 7 天

## 🆕 v5.2 前端 L0 缓存层（IndexedDB）

**设计目标**：解决页面刷新后全量重新加载的问题，实现增量更新

```
┌─────────────────────────────────────────────────────────────┐
│                    前端缓存层 (新增)                        │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ 🆕 L0 IndexedDB (MyQuantKlineDB)                    │   │
│  │                                                      │   │
│  │  容量: 100只 × 5周期 ≈ 5MB                          │   │
│  │  过期: LRU + 7天                                     │   │
│  │  延迟: <10ms 本地读取                                │   │
│  │                                                      │   │
│  │  场景:                                               │   │
│  │  • 页面刷新后从本地恢复                              │   │
│  │  • 增量更新（只获取1-10条新数据）                    │   │
│  │  • 减少HTTP传输 99%                                  │   │
│  └─────────────────────────────────────────────────────┘   │
│                           ↓ 未命中                          │
│  ┌─────────────────────────────────────────────────────┐   │
│  │ HTTP API /api/v5/kline/realtime/{symbol}           │   │
│  │ HTTP API /api/v5/kline/incremental/{symbol}        │   │
│  │ WebSocket /ws/kline/{symbol} (只推1分钟线)          │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

**性能提升**：
- 页面刷新：从 5000条全量 → 1-10条增量（99.8%↓）
- 传输数据量：从 ~200KB → ~1KB（99.5%↓）
- 加载时间：从 ~500ms → ~50ms（90%↓）

## 📊 数据流向（必须遵守）

```
前端 → API层 → Service层 → Adapter层 → 数据源
```

**禁止跳层**：
- ❌ 前端直接调用 Adapter
- ❌ API 层直接调用 Adapter
- ❌ Service 层直接实例化 Adapter（通过 `get_adapter()` 获取）

## 🏗️ 适配器分类

### 在线数据源适配器 → 连接 Online
| 适配器 | 说明 | 可用时间 |
|--------|------|----------|
| `tdxquant_adapter` | 通达信量化 SDK | 交易时间 |
| `xtquant_adapter` | 迅投 QMT SDK | 交易+盘后 |
| `pytdx_adapter` | 通达信服务器 | 24/7 |

### 本地缓存适配器 → 连接 HotDB/LocalDB
| 适配器 | 说明 | 数据类型 |
|--------|------|----------|
| `hotdb_adapter` | 热数据库 | 最近数据，快速访问 |
| `localdb_adapter` | 冷数据库 | 历史数据，QLib 格式 |

**❌ 错误**：把在线适配器放在缓存层
**✅ 正确**：在线适配器 → Online，本地适配器 → HotDB/LocalDB

## 🔗 服务调用链

```
HTTP API  →  SeamlessKlineService.get_kline()
                    │
                    ├─→ KlineService.get_historical_kline()
                    │         │
                    │         ├─→ hotdb_adapter → HotDB
                    │         ├─→ localdb_adapter → LocalDB
                    │         └─→ tdxquant/pytdx/xtquant_adapter → Online
                    │
                    └─→ AdjustmentFactorService.get_factor_table()

WebSocket → RealtimeMarketService → KlineService → Adapter → 数据源

HotDBService.smart_update()
        │
        └─→ KlineService.get_historical_kline()  ← 统一入口
                    │
                    ├─→ hotdb_adapter → HotDB
                    ├─→ localdb_adapter → LocalDB
                    └─→ tdxquant/pytdx/xtquant_adapter → Online
```

**关键规则**：
- 所有数据获取都通过 **KlineService** 统一入口
- HotDBService 在线补全 → KlineService → Adapter
- 禁止服务直接连接特定适配器（由 KlineService 路由决定）

## 🎯 数据源速查

| 场景 | 首选 | 备选 | 说明 |
|------|------|------|------|
| L1快照(盘中) | TdxQuant (0.6ms) | XtQuant | 最快 |

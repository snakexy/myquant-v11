# Online Serving模块概述

> **QLib模块**: Online Serving - 在线模型服务系统
> **阶段**: 跨三阶段（Research/Validation/Production）
> **最后更新**: 2026-02-23

---

## 🎯 模块定位

### QLib官方定义
> **Online Serving**是一个完整的在线模型服务系统，包括OnlineManager（总控制器）、OnlineStrategy（在线策略基类）、RollingStrategy（滚动策略）、OnlineTool（模型状态管理）和Updater（数据增量更新器）。

### 在三阶段中的位置
- **Research阶段**: 在线因子服务（P2可选）- 大规模因子计算和缓存
- **Validation阶段**: 历史模拟+模拟盘验证（P1核心）- 模拟在线服务环境
- **Production阶段**: 在线模型管理（P0必需）- 实盘模型管理、实时预测、信号准备

### 与其他模块的关系
- **依赖**: ML模型训练模块、数据管理模块
- **协作**: Meta Controller模块、RL模块
- **下游**: 交易执行系统、风险控制系统

### 在各阶段的核心价值
- **Research**: 提供大规模因子计算能力，加速研究迭代
- **Validation**: 提供模拟在线服务环境，验证策略有效性
- **Production**: 实现例行更新流程，保持模型最新状态

## 📊 核心功能

### QLib五大核心组件

#### 1. OnlineManager ⭐
- **功能**: 总控制器，管理在线模型和策略
- **职责**:
  - 管理在线模型生命周期
  - 协调数据更新和模型训练
  - 控制在线策略执行
  - 管理模型状态标签
- **API**: `online_manager.first_train()`, `online_manager.routine()`

#### 2. OnlineStrategy ⭐
- **功能**: 在线策略基类
- **实现类型**:
  - `RollingStrategy`: 滚动策略（最常用）
  - 自定义策略：继承基类实现
- **职责**:
  - 定义训练任务
  - 准备在线模型
  - 设置在线标签

#### 3. OnlineTool ⭐
- **功能**: 模型状态管理工具
- **职责**:
  - 设置在线标签（`online`/`offline`/`finetune`）
  - 查询模型状态
  - 切换在线模型

#### 4. Updater ⭐
- **功能**: 数据增量更新器
- **职责**:
  - 更新预测数据到最新
  - 增量更新特征数据
  - 维护数据一致性

#### 5. Signal准备 ⭐
- **功能**: 交易信号准备服务
- **职责**:
  - 为交易系统准备下一个交易周期的信号
  - 生成预测结果
  - 提供信号置信度

### 输入输出

**输入**:
- 训练好的ML/DL模型
- 历史数据
- 实时数据流
- 策略配置

**输出**:
- 在线模型状态
- 实时预测结果
- 交易信号
- 模型性能监控

## 📈 跨阶段应用

### Research阶段 - 在线因子服务（P2）

**核心价值**: 提供大规模因子计算能力

**主要功能**:
- 在线因子计算（处理大规模因子库）
- 因子缓存服务（加速重复计算）
- 实时因子更新

**连接节点**:
- `OS_R_Calc`: 在线因子计算
- `OS_R_Cache`: 因子缓存服务
- `OS_R_Update`: 实时因子更新

**API示例**:
```yaml
- POST /api/v1/research/online/calc
- GET /api/v1/research/online/cache/status
- POST /api/v1/research/online/update
```

### Validation阶段 - 模拟在线服务（P1 核心）

**核心价值**: 模拟实盘环境，验证策略有效性

**主要功能**:
- 历史模拟模式：在历史数据上模拟在线服务
- 模拟盘服务：模拟实盘交易环境
- 性能监控：监控模拟盘表现

**连接节点**:
- `OS_V_Sim`: 历史模拟服务
- `OS_V_Paper`: 模拟盘服务
- `OS_V_Monitor`: 性能监控

**API示例**:
```yaml
- POST /api/v1/validation/online/sim/start
- GET /api/v1/validation/online/paper/status
- GET /api/v1/validation/online/monitor/performance
```

### Production阶段 - 在线模型管理（P0 必需）

**核心价值**: 实盘环境的完整在线模型管理系统

**主要功能**:
- 实盘模型管理：部署、切换、退役、性能监控
- 实时预测服务：为交易系统提供实时预测
- 例行更新流程：每日/每周自动routine
  - 数据更新→任务准备→模型训练→模型切换→信号准备
- 信号准备服务：为交易系统准备下一个交易周期信号

**连接节点**:
- `OS_P_Deploy`: 模型部署管理
- `OS_P_Predict`: 实时预测服务
- `OS_P_Signal`: 信号准备服务
- `OS_P_Routine`: 例行更新流程

**API示例**:
```yaml
- POST /api/v1/production/online/deploy
- GET /api/v1/production/online/predict
- POST /api/v1/production/online/signal/prepare
- POST /api/v1/production/online/routine/trigger
```

## 🏗️ 技术架构

### 核心技术
- **后端**: Python + FastAPI
- **框架**: QLib Online Serving
- **数据存储**: .pkl格式（符合项目架构）
- **异步处理**: asyncio（Production阶段）

### 存储路径
```
backend/data/online_serving/
├── models/              # 在线模型存储
├── signals/             # 预测信号存储
├── logs/                # 在线服务日志
├── snapshots/           # 模型快照
└── state/               # 模型状态文件
```

### 典型工作流

#### Research阶段：因子计算服务
```python
# 1. 创建在线计算服务
online_service = FactorCalculationService()

# 2. 计算因子
result = online_service.calculate_factor(
    factor_name="alpha158",
    start_date="2020-01-01",
    end_date="2023-12-31"
)

# 3. 缓存结果
online_service.cache_result(result)
```

#### Production阶段：例行更新流程
```python
# 每日例行更新
online_manager.routine(
    cur_time=today,
    task_kwargs={...},
    model_kwargs={...},
    signal_kwargs={...
})
# routine内部自动完成：
# 1. 更新预测到今天（Updater）
# 2. 准备新训练任务（OnlineStrategy）
# 3. 训练模型
# 4. 准备在线模型（OnlineStrategy）
# 5. 设置在线标签（OnlineTool）
# 6. 准备明天的交易信号
```

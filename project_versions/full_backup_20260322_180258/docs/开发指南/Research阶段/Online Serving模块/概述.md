## 🎯 模块定位

### 重要概念
> **Online Serving ≠ "在线滚动训练"**
>
> Online Serving 是一个**完整的在线模型服务系统**，不仅仅是滚动训练。

### 核心组件

| 组件 | 说明 |
|------|------|
| **OnlineManager** | 总控制器，管理在线模型和策略 |
| **OnlineStrategy** | 在线策略基类 |
| **RollingStrategy** | 滚动策略（仅是其中一种策略） |
| **OnlineTool** | 在线模型状态管理工具 |
| **Updater** | 数据增量更新器 |

### 在三阶段的位置

| 阶段 | 功能 | 优先级 |
|------|------|--------|
| **Research** | 在线因子服务 | P2 |
| **Validation** | 历史模拟+模拟盘验证 | P1 核心 |
| **Production** | 实时预测+信号准备 | P0 必需 |

## 📊 Research阶段 - 在线因子服务

### 功能定位
- **集成位置**: 因子分析步骤
- **核心价值**: 为IC/IR计算器提供高性能因子计算和缓存服务

### 核心功能

#### 1. 在线因子计算（大数据场景）
- **场景**: 计算 4000股票 × 360因子 = 144万数据点
- **方案**: 分布式/并行计算
- **优化**: 增量计算，只计算变化的部分

#### 2. 因子缓存服务
- **目的**: 避免重复计算
- **策略**: LRU缓存 + 时间过期
- **存储**: Redis / Memory Dict

#### 3. 实时因子更新
- **触发**: 新交易日数据到达
- **方式**: Updater增量更新
- **效率**: 只更新变化的数据

### 使用场景

```
场景: IC/IR计算需要因子数据

流程:
1. IC/IR计算器请求因子数据
2. Online Serving 检查缓存
   - 缓存命中: 直接返回
   - 缓存未命中: 计算并缓存
3. 返回因子数据给IC/IR计算器
4. IC/IR计算器进行指标计算

优势:
- 避免重复计算
- 快速响应
- 统一数据源
```

## 📝 API端点设计

### Research阶段 API

| 端点 | 方法 | 功能 | 说明 |
|------|------|------|------|
| `/api/v1/research/online/health` | GET | 健康检查 | 服务状态检查 |
| `/api/v1/research/online/calculate` | POST | 在线因子计算 | 大规模因子计算 |
| `/api/v1/research/online/cache` | GET | 获取缓存因子 | 从缓存获取因子数据 |
| `/api/v1/research/online/cache` | POST | 缓存因子数据 | 保存因子到缓存 |
| `/api/v1/research/online/incremental` | POST | 增量更新 | Updater增量更新 |
| `/api/v1/research/online/status` | GET | 服务状态 | 缓存命中率等统计 |

## 🔗 与因子分析步骤的集成

### 集成方式
在线因子服务作为 **因子分析步骤 (Step 3)** 的一部分，提供因子数据的计算和缓存能力。

### 前端界面位置
```
Research 页面
  └── Step 3: 因子分析
        ├── 概览面板
        ├── IC/IR分析
        ├── 分布分析
        ├── 相关性分析
        └── 在线因子服务 ← 新增面板
```

### 使用流程
1. 用户进入因子分析步骤
2. 点击"在线因子服务"面板
3. 配置需要计算的因子和股票范围
4. 点击"获取因子数据"
5. 系统自动计算/缓存因子数据
6. 因子数据可用于IC/IR分析

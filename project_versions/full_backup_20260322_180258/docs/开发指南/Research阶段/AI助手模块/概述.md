## 🎯 模块定位

### 在三阶段中的位置
- **Research阶段**: 提供AI驱动的智能因子生成和对话辅助功能
- **核心价值**: 通过DeepSeek大模型实现智能因子生成、多轮对话管理、数据持久化

### v2.0 重大升级
从v1.0（模拟数据）升级到v2.0（生产级系统）：

| 功能 | v1.0 | v2.0 | 提升 |
|--------|-------|-------|------|
| AI生成 | 模拟数据 | DeepSeek真实API | ⭐⭐⭐ |
| 对话管理 | 内存列表 | ContextManager + 数据库 | ⭐⭐⭐ |
| 配置管理 | JSON文件 | 数据库表（ai_config） | ⭐⭐⭐ |
| 因子保存 | 不支持 | ai_generated_factors表 | ⭐⭐⭐ |
| 数据源 | 返回None | 3源优先级加载 | ⭐⭐ |

### 与其他模块的关系
- **上游**:
  - 数据管理模块 - 股票数据查询
  - 因子计算模块 - 因子数据查询
  - 因子分析模块 - 分析结果查询
- **下游**: 无（辅助模块，为用户提供AI交互界面）

## 📊 核心功能

### 功能列表

#### 1. AI因子生成 ⭐
- **真实AI生成**: 使用DeepSeek大模型生成因子
- **多种因子类型**: 动量、反转、波动率等
- **因子表达式**: 自动生成QLib/Python表达式
- **参数建议**: 提供参数范围和默认值
- **历史持久化**: 生成的因子保存到数据库

#### 2. 多轮对话管理 ⭐ v2.0新增
- **会话创建**: 带标题和用户目标的会话
- **消息管理**: 用户和AI消息轮次保存
- **会话切换**: 支持多会话并行，相关度计算
- **上下文保持**: 最近N条消息的上下文窗口
- **Token限制**: 单会话最多2000 tokens

#### 3. 数据持久化 ⭐ v2.0新增
- **配置管理**: API密钥、模型参数（ai_config表）
- **对话历史**: 完整对话记录（ai_conversation_history表）
- **因子库**: AI生成因子归档（ai_generated_factors表）
- **外键关联**: 因子表关联对话历史

#### 4. 数据源集成 ⭐ v2.0新增
- **因子数据**: 3源优先级（QLib → DB → Legacy）
- **收益率数据**: 从K线数据计算
- **多因子数据**: 批量加载和合并
- **智能降级**: 数据源不可用时优雅降级

#### 5. 配置管理
- **API密钥管理**: 支持加密存储
- **模型参数配置**: temperature、max_tokens等
- **系统配置**: 历史保留天数、上下文条数等

### 输入输出

**输入**:
- 自然语言问题（因子生成需求）
- 会话管理请求（创建、切换、查询）
- 配置管理请求（保存、查询）

**输出**:
- AI生成的因子代码和表达式
- 对话历史记录
- 配置查询结果

### 核心技术
- **后端**: Python 3.10+ + FastAPI
- **AI服务**: DeepSeek API（https://api.deepseek.com/v1）
- **数据库**: MySQL 8.0+（myquant_ai库）
- **ORM**: SQLAlchemy（3张表）
- **异步**: httpx.AsyncClient
- **重试**: tenacity（指数退避）

## 🚀 快速开始

### 1. 数据库初始化
```bash
python backend/database/init_ai_assistant_db.py
```

### 2. 配置API密钥
```bash
curl -X POST http://localhost:8000/api/v1/research/ai/config/save \
  -H "Content-Type: application/json" \
  -d '{"config_key": "deepseek_api_key", "config_value": "your-api-key"}'
```

### 3. 测试健康检查
```bash
curl http://localhost:8000/api/v1/research/ai/health
```

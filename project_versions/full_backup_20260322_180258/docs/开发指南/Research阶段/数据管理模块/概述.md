## 📊 核心功能

### 功能列表

#### 1. 数据获取 (Data Acquisition)
- **L0-L5数据类型支持**:
  - L0: 实时行情监控（WebSocket订阅推送）
  - L1: 实时快照（最新1根K线）
  - L2: 历史快照（30天聚合统计）
  - L3: 完整K线数据（所有周期）
  - L4: 财务数据
  - L5: 板块数据
- **三级存储架构**:
  - 实时层: Redis缓存（3秒过期）
  - 历史层: SQLite持久化
  - 量化层: Qlib格式（从SQLite转换）
- **智能数据源切换**: 根据data_level自动选择最优数据源
- **数据统一接口**: 通过DataManager屏蔽底层差异

#### 2. 数据清洗 (Data Cleaning)
- **缺失值处理**:
  - 前向填充 (forward fill)
  - 后向填充 (backward fill)
  - 均值/中位数填充
  - 删除含缺失值的行
- **异常值检测与处理**:
  - IQR方法 (四分位距)
  - Z-score方法
  - 钳制处理 (clipping)
- **重复数据处理**: 自动识别并去除重复记录
- **数据类型标准化**: 统一数据类型和格式

#### 3. 数据质量管理 (Data Quality Management)
- **质量评估**: 计算数据质量评分（0-100分）
- **质量检查**: 缺失值、重复值、异常值检测
- **质量报告**: 生成详细的数据质量报告
- **问题诊断**: 识别数据中的问题并提供修复建议

#### 4. 数据预处理 (Data Preprocessing)
- **特征标准化**:
  - Z-score标准化
  - Min-Max标准化
  - 鲁棒标准化 (RobustScaler)
- **数据转换**: 类型转换、格式统一
- **数据排序**: 按时间戳排序

#### 5. 数据导出 (Data Export)
- 支持多种格式：CSV、Excel、Parquet、JSON
- 自定义导出参数（时间范围、字段选择）
- 批量导出功能

#### 6. 股票信息管理 (Stock Information)
- 股票列表查询（支持筛选、排序、分页）
- 股票详细信息（基本信息、财务数据、技术指标）
- 智能搜索（按代码、名称、拼音首字母）

#### 7. 数据库管理 (Database Management)
- 数据库统计信息（股票数量、时间范围、数据大小）
- 分类统计（按行业、概念、板块）
- 服务状态监控（Redis、QLib、TDX）
- 数据源配置管理

### 输入输出

**输入**:
- K线数据（L0-L5分层）
- 股票基础信息
- 数据源配置

**输出**:
- 清洗后的数据
- 统计分析报告
- 导出文件

## 🏗️ 技术架构

### 文件结构
```

├── services/research/

│   ├── data_management_service.py    (~930行) - 统一数据管理服务（含缓存+异步）
│   ├── data_cache_service.py         (~260行) - Redis缓存适配器
│   ├── stock_search.py               (~430行) - 股票搜索引擎（拼音+缓存）
│   └── stock_service.py              (~640行) - 股票信息管理（含搜索+缓存）
├── database/
│   ├── schema/                       - 数据库DDL脚本
│   │   ├── data_source_config.sql
│   │   ├── index_constituents.sql
│   │   └── data_refresh_task.sql
│   ├── models/
│   │   └── data_management_models.py - ORM模型
│   └── init_data_management_db.py    - 初始化脚本
├── tests/
│   ├── database/
│   │   └── test_data_management_models.py
│   └── services/research/
│       └── test_data_cleaning_service.py
├── test_tdd_async.py                 (~184行) - TDD异步方法测试
├── test_pandas_fillna.py             (~134行) - Pandas兼容性测试
├── api/v1/research/
│   └── data_router.py                (~430行) - API路由
└── models/
    └── 数据模型定义

frontend/
└── src/views/research/
    └── DataManagementView.vue        (~1198行)
```

### 核心技术
- **后端**: Python + FastAPI
- **前端**: Vue 3 + TypeScript + Vite
- **数据处理**: NumPy + Pandas
- **数据存储**: Redis + QLib + 文件系统

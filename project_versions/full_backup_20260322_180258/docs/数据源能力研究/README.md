# 数据源能力研究

## 目的

在深入开发之前，**彻底研究清楚**各个数据源的能力、限制、获取方式。

## 研究对象

1. **XtQuant (迅投QMT)**
2. **TdxQuant (通达信量化)**
3. **PyTdx (通达信)**
4. **本地数据库**

## 研究维度

对每个数据源，研究以下内容：

### 1. API能力
- 支持哪些数据类型（K线、分笔、财务等）
- 支持哪些周期（1m, 5m, 1d等）
- 支持哪些复权方式
- 在线获取 vs 本地下载

### 2. 参数限制
- 时间范围限制（16天等）
- 数量限制（count参数）
- 参数类型（字符串 vs 数字）
- 必需参数 vs 可选参数

### 3. 数据格式
- 返回格式（DataFrame, dict等）
- 列名和索引
- 时间格式
- 空值处理

### 4. 性能特征
- 响应时间
- 并发能力
- 缓存机制

### 5. 实际测试
- 测试脚本
- 测试结果
- 边界情况
- 常见错误

## 研究原则

1. **测试优先** - 用实际测试验证，不依赖文档
2. **记录事实** - 记录实测结果，不是猜测
3. **边界测试** - 测试极限情况和错误场景
4. **参数敏感** - 详细记录每个参数的影响

## 工作原理

### 数据共享机制

```
┌─────────────────────────────────────────────────────────────┐
│ 共享数据目录                                                │
│ E:\迅投xt量化交易终端 miniQMT独立版\userdata_mini\datadir   │
│                                                             │
│ ├── block/          # 板块分类数据                           │
│ ├── index/          # 指数成分股权重                         │
│ ├── kline/          # K线数据                               │
│ └── tick/           # Tick数据                              │
└─────────────────────────────────────────────────────────────┘
                          ↑
          ┌───────────────┴───────────────┐
          │                               │
    ┌─────┴─────┐                   ┌─────┴─────┐
    │ 实例1 (SH) │                   │ 实例2 (SZ) │
    │ 8887080618 │                   │ 18049905250│
    │ 300订阅    │                   │ 300订阅    │
    └────────────┘                   └────────────┘
```

### 工作流程

1. **数据下载**（只需一次）
   - 在任一QMT客户端中下载板块、指数数据
   - 数据存储到共享目录
   - 两个实例都能访问

2. **订阅分配**（自动）
   - 上海市场股票 → 实例1
   - 深圳市场股票 → 实例2
   - 总订阅能力：600只

3. **数据读取**（共享）
   - 两个实例从同一目录读取历史数据
   - 避免重复下载
   - 保证数据一致性

## 使用说明

### 首次使用

1. **下载数据**
   ```bash

   python "e:\MyQuant_v10.0.0\docs\数据源能力研究\XtQuant\test_107_download_and_verify.py"
   ```

3. **重启服务**
   - 重启后端服务以加载新配置
   - 确认两个实例都正常连接

### 日常维护

1. **定期更新数据**
   - 板块数据：每周一次

   - 共享数据目录大小
   - 定期清理过期数据

## 注意事项

### ⚠️ 权限问题

确保两个QMT客户端都有权限访问共享目录：
- 读取权限：必须
- 写入权限：必须（用于下载和更新）

### ⚠️ 文件锁定

如果遇到文件访问错误：
1. 确保只有一个QMT客户端在写入数据
2. 避免同时下载大量数据

### ⚠️ 数据一致性

- 两个实例读取同一数据源，天然一致
- 更新数据后两个实例都能看到

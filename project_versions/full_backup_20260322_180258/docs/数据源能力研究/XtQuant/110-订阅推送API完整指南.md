## 核心API

### subscribe_quote() - 订阅行情数据

```python
xtdata.subscribe_quote(
    stock_code,     # ⭐ 股票代码（必填）
    period='1d',     # 周期（默认'1d'）
    start_time='',   # 开始时间（可选）
    end_time='',     # 结束时间（可选）
    count=0,         # 数量（默认0）
    callback=None    # 回调函数（可选）
)
```

**返回值**:
- 成功：返回订阅号（>0）
- 失败：返回-1

---

## 参数详解

### stock_code - 股票代码

**格式**：
- 上海股票：`600000.SH`
- 深圳股票：`000001.SZ`
- 全市场：`'SH'` 或 `'SZ'`

**示例**：
```python
# 订阅单只股票
xtdata.subscribe_quote(stock_code='600519.SH', period='tick')

# 订阅上海全市场
xtdata.subscribe_quote(stock_code='SH', period='1m')
```

---

### period - 周期类型

| 值 | 说明 | 适用场景 |
|----|------|---------|
| `'tick'` | 分笔数据 | 实时分时图 |
| `'1m'` | 1分钟K线 | 短周期K线 |
| `'5m'` | 5分钟K线 | 中周期K线 |
| `'1d'` | 日K线 | 长周期K线 |

**注意**：
- ⚠️ `tick` 数据量大，谨慎使用
- ✅ K线数据更常用

---

### callback - 回调函数

**函数签名**：
```python
def callback(data):
    """
    data: {
        '600519.SH': {...},  # 股票代码: 行情数据
        '000001.SZ': {...}
    }
    """
    pass
```

**注意事项**：
- ⚠️ 回调在主线程执行，避免耗时操作
- ⚠️ 推送频率约3秒
- ⚠️ 必须配合 `xtdata.run()` 使用

---

## run() - 阻塞线程接收推送

```python
xtdata.run()
```

**作用**：
- 阻塞当前线程
- 持续接收推送数据
- 触发callback函数

**重要**：
- ⚠️ **必须调用**，否则收不到推送
- ⚠️ 会永久阻塞，需要在独立线程中运行
- ⚠️ 程序结束时自动退出

## 常见问题

### Q1: 收不到推送？

**可能原因**：
1. **非交易时间** - 周末、节假日、收盘后
2. **未调用run()** - 必须调用才能接收
3. **参数错误** - 检查stock_code格式

**解决方法**：
```python
# 1. 检查交易时间
from datetime import datetime

def is_trading_time():
    """判断是否交易时间（包含集合竞价）"""
    now = datetime.now()

    # 周末
    if now.weekday() >= 5:
        return False

    hour, minute = now.hour, now.minute
    time_min = hour * 60 + minute

    # 集合竞价: 9:15-9:25
    if 555 <= time_min <= 565:
        return True

    # 上午连续: 9:30-11:30
    if 570 <= time_min <= 690:
        return True

    # 下午连续: 13:00-15:00
    if 780 <= time_min <= 900:
        return True

    return False

if not is_trading_time():
    print("当前是非交易时间，收不到推送")
    print("交易时间: 9:15-11:30, 13:00-15:00")

# 2. 确保调用了run()
threading.Thread(target=xtdata.run, daemon=True).start()

# 3. 检查参数格式
# 正确：'600519.SH'
# 错误：'600519'
```

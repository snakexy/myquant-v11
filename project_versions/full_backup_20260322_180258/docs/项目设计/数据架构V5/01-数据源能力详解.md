## 1.1 TdxQuant - 交易时间王者 🏆

### 核心优势

| 能力 | 性能 | 数据质量 | 地位 |
|------|------|----------|------|
| **L1 实时快照** | **0.60ms** | 高 | 交易时间最快 |
| **板块数据** | **6.99ms** | **极高** | **586个板块，绝对优势** |
| 历史K线 | ~50ms | 高 | 完整历史 |
| 财务数据 | ~30ms | 高 | 完整 |
| 订阅功能 | 5-6ms轮询 | 高 | 100股上限 |

### 板块数据详解

**双层数据源架构**：
```
┌─────────────────────────────────┐
│     TdxQuant 板块数据层          │
├─────────────────────────────────┤
│  第一层：通达信服务器             │
│  - 实时板块分类                  │
│  - 成分股列表                    │
│  - 板块指数行情                  │
├─────────────────────────────────┤
│  第二层：本地通达信数据           │
│  - 缓存板块信息                  │
│  - 快速读取                      │
│  - 离线可用                      │
└─────────────────────────────────┘
```

**586个板块包括**：
- **行业板块**（150+）：申万一级行业、二级行业
- **概念板块**（400+）：新能源、人工智能、半导体等
- **地区板块**（50+）：各省板块、长三角、珠三角

### 关键限制 ⚠️

| 限制 | 说明 | 影响 |
|------|------|------|
| **❌ 不支持当天分钟线** | API限制 | 当天分钟必须用XtQuant/PyTdx2 |
| **仅交易时间可用** | 依赖通达信终端 | 非交易时间无法使用 |
| 订阅上限100股 | 单连接限制 | 大量订阅需多连接 |

### 最佳使用场景

✅ **强烈推荐**：
- 实时快照监控（0.60ms最快）
- 板块轮动分析（6.99ms，586板块）
- 热点板块发现
- 交易时间高频数据获取

❌ **不推荐**：
- 当天分钟线数据
- 非交易时间数据获取

## 1.2 XtQuant - 当天分钟线专家

### 核心优势

| 能力 | 性能 | 数据质量 | 地位 |
|------|------|----------|------|
| **当天分钟线** | **0.90ms缓存** | 高 | **交易时间最优** |
| L1 实时快照 | 720ms下载 + **6ms缓存** | 高 | 批量获取快 |
| **订阅功能** | **<1ms推送** | 高 | **300股上限** |
| 历史K线 | ~500ms | 中 | 在线获取有限制，长期回测建议先下载 |

### 订阅功能详解

```python
from xtquant import xtdata

# 订阅300只股票
symbols = ["600000.SH", "000001.SZ", ...]  # 最多300只
xtdata.subscribe_quote(symbols, callback=on_quote_update)

# 推送延迟：<1ms
# 推送机制：事件驱动，有变化才推送
```

### 历史K线获取能力（⚠️ 重要说明）

**`get_market_data_ex` 的真实行为**：
- ⚠️ **读取QMT本地缓存**，并非真正的"在线获取"
- 数据范围取决于QMT本地数据库的存储情况
- **获取当天数据**：需在交易时间让QMT自动更新，或先调用`download_history_data()`

**验证测试**：详见 [验证测试-get_market_data_ex数据时效性.md](../../../数据源能力研究/XtQuant/验证测试-get_market_data_ex数据时效性.md)
- 测试时间：2026-03-20 04:52
- 最新数据：2026-03-19 15:00（昨天收盘）
- 证明 `get_market_data_ex` 只能读取本地缓存

| 周期 | 本地缓存读取 | 耗时 | 说明 |
|------|-------------|------|------|
| 日K | 750条 (3年) | 9ms | ✅ K线图完全够用 |
| 5分钟K | 5532条 (约5个月) | 15ms | ⚠️ 本地缓存上限 |
| 1分钟K | 2500条 (约2周) | 12ms | ⚠️ 本地缓存上限 |

### 关键限制 ⚠️

| 限制 | 说明 | 影响 |
|------|------|------|
| **❌ 不支持板块数据** | 功能限制+代码映射不兼容 | 板块业务必须用TdxQuant |
| **读取本地缓存** | `get_market_data_ex`读取QMT本地数据，非真正在线 | 获取当天数据需确保QMT已更新 |
| **数据量限制** | 本地缓存5分钟K上限5532条 | 长期回测需先下载 |
| 仅交易+盘后可用 | 依赖QMT | 周末/节假日无法使用 |

### 最佳使用场景

✅ **强烈推荐**：
- 当天分钟线数据（0.90ms最优）
- L0订阅缓存（300股，<1ms推送）
- K线图展示（本地缓存读取，9-15ms）
- 盘后时间个股数据（QMT运行时）

❌ **不推荐**：
- **板块相关业务**（不支持+代码映射不兼容）
- 非交易时间数据获取（QMT未运行时）

## 1.3 PyTdx2 - 非交易时间救星 🌟

### 核心优势

| 能力 | 性能 | 数据质量 | 地位 |
|------|------|----------|------|
| **24/7可用性** | - | - | **非交易时间唯一在线数据源** |
| L1 实时快照 | **10-19ms** | 高 | 可用 |
| 当天分钟线 | **10-19ms** | 高 | 可用 |
| 历史K线 | **~19ms** | 高 | 可用 |
| 板块数据 | **~15ms** | 中 | 部分板块 |
| 财务数据 | ~20ms | 中 | 37个字段 |

### 四大优化

1. **动态服务器选速**
   - 实时测试113个服务器
   - 性能差异：最快12ms vs 最慢41ms = **3.4x**

2. **Redis缓存**
   - 除权信息缓存
   - 缓存命中：<1ms

3. **异步支持**
   - AsyncPyTdxAdapter
   - FastAPI场景5-10x提升

4. **连接池**
   - 多连接复用
   - 大批量并发3x提升

### 关键限制 ⚠️

| 限制 | 说明 | 影响 |
|------|------|------|
| 响应时间较慢 | 10-19ms | 比TdxQuant慢 |
| 板块数量有限 | 部分板块 | 不如TdxQuant的586个 |

### 最佳使用场景

✅ **强烈推荐**：
- **非交易时间数据获取**（唯一选择）
- 当天分钟线（非交易时间）
- 异步场景（FastAPI）

❌ **不推荐**：
- 追求极致性能（TdxQuant 0.60ms更快）
- 板块业务（TdxQuant 586板块更全）

## 1.4 QLib - 本地数据库

### 核心优势

| 能力 | 性能 | 数据质量 | 地位 |
|------|------|----------|------|
| **历史K线** | **7-10ms** | 极高 | **历史数据首选** |
| 当天分钟线 | **7-10ms** | 高 | 本地读取 |
| ❌ L1实时快照 | - | - | 不支持 |
| ❌ 板块数据 | - | - | 不支持 |

### 数据存储格式

```
data/qlib_data/
├── stocks/
│   ├── 600000/
│   │   ├── day.bin         # 日线数据
│   │   ├── min1.bin        # 1分钟数据
│   │   ├── min5.bin        # 5分钟数据
│   │   └── ...
│   └── ...
└── ...
```

### 最佳使用场景

✅ **强烈推荐**：
- 历史K线查询（7-10ms最快）
- 回测系统
- 数据归档存储

❌ **不推荐**：
- 实时快照
- 板块数据

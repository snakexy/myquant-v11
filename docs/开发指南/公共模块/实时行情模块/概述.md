## 🎯 模块定位

### 核心价值

实时行情模块是 MyQuant 的**看盘工具**，提供 K 线图表、实时报价、五档盘口显示。是所有需要观察市场数据的场景的基础能力。

**本质：看盘用的 K 线展示工具，不是实时监控系统。**

### 与其他模块的关系

- **上游**: 场景服务层 (`data_providers/services/realtime_market/`) — API 层只调 Service，不直接调适配器
- **Service 层上游**: V5 数据适配器（TdxQuant/XtQuant/PyTdx/LocalDB）— Service 层编排适配器调用
- **下游**: 所有需要行情数据的模块（回测验证、模拟实盘、实盘交易）
- **平级**: 板块数据模块、自选股管理（待建）

## 📊 核心功能

### 功能列表

#### 1. K 线图表
- 支持 7 种周期：1m、5m、15m、30m、1h、1d、1w
- 中国股市配色（红涨绿跌）
- 日 K 线自动填充周末/节假日空白
- 成交量柱状图

#### 2. 实时报价
- 当前价格、涨跌额、涨跌幅
- 今日开盘、最高、最低
- 成交量、成交额
- 动态刷新：交易时段 3-5 秒、非交易时段 30 秒

#### 3. 五档盘口
- 买一~买五（价格 + 量）
- 卖一~卖五（价格 + 量）

#### 4. 市场状态判断
- 11 个时段识别（开盘前、集合竞价、上午/下午交易、午休、收盘后、周末）
- 根据时段动态选择数据源（交易时间用 TdxQuant/XtQuant，盘后用 PyTdx）
- 根据时段动态调整刷新频率和缓存策略

### 输入输出

**输入**:
- 股票代码（如 `600000.SH`）
- K 线周期（1m/5m/15m/30m/1h/1d/1w）

**输出**:
- K 线数据数组（时间戳、开高低收、量、颜色）
- 实时快照（价格、涨跌、盘口）
- 数据源标识（TdxQuant/XtQuant/PyTdx/LocalDB）

## 🔗 依赖关系

### 依赖哪些模块

| 模块 | 用途 | 文件路径 |
|------|------|---------|
| 场景服务层 | 业务编排、缓存 | `backend/data_providers/services/realtime_market/` |
| V5 适配器层 | 数据获取 | `backend/data_providers/adapters/v5/` |
| 路由选择器 | 数据源选择 | `backend/data_providers/routing/selector.py` |
| V5 数据模型 | 统一数据格式 | `backend/data_providers/models/` |
| 交易时间检测器 | 开盘/收盘状态判断 | `backend/data/utils/trading_time.py` |

**调用链**: `API路由` → `RealtimeMarketService` → `TradingTimeChecker` + `SourceSelector` → `V5DataAdapter` → 数据源

### 依赖哪些外部系统

| 系统 | 用途 | 备注 |
|------|------|------|
| TdxQuant SDK | 实时快照（0.60ms） | 交易时间，不支持当日分钟线 |
| XtQuant SDK | 当日分钟线（0.90ms） | 交易时间，不支持板块 |
| PyTdx | 24/7 全功能（10-19ms） | 非交易时间唯一在线源 |
| LocalDB (QLib) | 历史 K 线（7-10ms） | 本地数据库 |

### 被哪些模块依赖

- **历史回测模块** — 回测时对比实时行情
- **模拟实盘模块** — 模拟交易时获取行情
- **实时监控模块** — 盘中监控市场状态
- **实盘交易模块** — 交易信号需要实时数据

## 🏗️ 技术架构

### 架构原则：三层分离

```
┌─────────────────────────────────────────────────┐
│  API 层 (routes)                                 │
│  backend/api/v1/quotes/                          │
│  只处理 HTTP 请求/响应，不做数据获取逻辑           │
├─────────────────────────────────────────────────┤
│  服务层 (service)                                │
│  backend/data_providers/services/realtime_market/ │
│  编排数据获取逻辑、缓存、业务规则                  │
├─────────────────────────────────────────────────┤
│  适配器层 (adapter)                              │
│  backend/data_providers/adapters/v5/             │
│  连接具体数据源，做数据格式归一化                   │
└─────────────────────────────────────────────────┘
```

### 数据源路由策略

| 数据类型 | 交易时间优先级 | 非交易时间 |
|---------|--------------|-----------|
| 实时快照 | TdxQuant (0.60ms) → XtQuant → PyTdx | PyTdx (唯一在线) |
| K 线（日线） | LocalDB (7-10ms) → XtQuant → PyTdx | LocalDB → PyTdx |
| K 线（分钟线） | XtQuant (0.90ms) → PyTdx | PyTdx (唯一在线) |

### 核心技术

| 层级 | 技术 | 说明 |
|------|------|------|
| 后端框架 | FastAPI | 异步 API |
| 日志 | loguru | 禁止 print |
| 数据适配器 | V5DataAdapter 基类 | 统一接口 |
| 前端框架 | Vue 3 + `<script setup lang="ts">` | TypeScript |
| 图表库 | lightweight-charts | TradingView 出品 |
| 数据刷新 | 5 秒轮询 | 非 WebSocket |

## 📋 实施概要

### 目标

将实时行情模块从"能跑但架构不合规"重构为"按 V5 设计文档正确实现"。

### 工作内容

- ⬜ 轮次1：后端重构 — API 接入 Service 层，移除 V4 代码，迁移 TradingTimeChecker
- ⬜ 轮次2：前端修正 — API 调用规范化，自选股持久化，动态刷新
- ⬜ 轮次3：验证测试 — 端到端数据流验证

### 核心原则

1. **三层分离**: API → Service → Adapter，每层职责清晰
2. **先文档后代码**: 每轮实施前先确认文档
3. **越界控制**: 只改计划内的文件
4. **V4 清零**: 彻底移除 unified_data_manager 依赖

## 🏗️ 轮次1：后端重构

### 目标

让 API 层通过 Service 层获取数据，不再直接调用适配器。清理 V4 代码。

### 步骤

#### 1.0 迁移 TradingTimeChecker 到 V5 路径

**源文件**: `backend/data/utils/trading_time.py`（V4 路径）
**目标**: `backend/data_providers/utils/trading_time.py`（V5 路径）

当前 `TradingTimeChecker` 在 V4 目录下，V5 的 Service 层不应依赖 V4 路径。

动作：
1. 复制 `trading_time.py` 到 `backend/data_providers/utils/`
2. 调整 import 路径
3. 后续 V5 代码统一从 `data_providers.utils.trading_time` 导入

#### 1.1 评估并接入 RealtimeMarketService

**文件**: `backend/data_providers/services/realtime_market/service.py`

当前问题：Service 已实现但 API 没有调用。

需要确认：
- `get_realtime_quotes()` 方法的数据格式是否与 API 的 `QuoteSnapshot` 兼容
- `get_kline()` 的调用链是否正确（它委托给 KlineService）
- 缓存逻辑是否合理

动作：如果 Service 接口设计合理 → 直接在 API 层调用；如果不合理 → 调整 Service 接口后接入。

#### 1.2 重构 snapshot.py — 接入 Service 层

**文件**: `backend/api/v1/quotes/snapshot.py`

**当前**: 直接 import 适配器并调用
```python
# ❌ 当前
from data_providers.adapters.v5.tdxquant_adapter import V5TdxQuantAdapter
quotes = adapter.get_quote(symbols)
```

**目标**: 通过 Service 层调用
```python
# ✅ 目标
from data_providers.services.realtime_market import RealtimeMarketService
service = RealtimeMarketService()
quotes = service.get_realtime_quotes(symbols)
```

#### 1.3 重构 kline.py — 接入 Service 层

**文件**: `backend/api/v1/quotes/kline.py`

**当前**: 直接 import 适配器
```python
# ❌ 当前
from data_providers.adapters.v5.localdb_adapter import V5LocalDBAdapter
klines = adapter.get_kline(symbols=[symbol], period=period, count=count)
```

**目标**: 通过 Service 层调用
```python
# ✅ 目标
from data_providers.services.realtime_market import RealtimeMarketService
service = RealtimeMarketService()
klines = service.get_kline(symbol, period, count)
```

#### 1.4 清理 market/routers.py 中的 V4 代码

**文件**: `backend/api/v1/market/routers.py`

问题行：
- L42: `from data.unified_data_manager_stocklist_ext import ...`
- L82: `from data.unified_data_manager import ...`
- L949, L1495: 同上

处理方式：
1. 列出所有使用 `unified_data_manager` 的端点
2. 评估哪些端点有功能价值
3. 有价值的 → 迁移到 V5 适配器（新建或合并到 quotes API）
4. 无价值的 → 直接删除

#### 1.5 Service 层方法签名确认

确保 `RealtimeMarketService` 暴露的方法与 API 层需求匹配：

| API 端点 | Service 方法 | 当前状态 |
|---------|-------------|---------|
| `GET /snapshot/` | `get_realtime_quotes(codes)` | ✅ 已有 |
| `GET /snapshot/{symbol}` | `get_realtime_quotes([symbol])` | ✅ 已有 |
| `GET /kline/{symbol}` | `get_kline(symbol, period, count)` | ✅ 已有 |
| `GET /market/status` | `TradingTimeChecker.get_current_phase()` | ⚠️ 需新增 |

### 完成标准

- ⬜ API 层不再直接 import 任何 V5 适配器
- ⬜ 所有数据获取通过 `RealtimeMarketService` 进行
- ⬜ `TradingTimeChecker` 迁移到 V5 路径 `data_providers/utils/`
- ⬜ 市场状态 API 端点正常工作
- ⬜ `market/routers.py` 中无 `unified_data_manager` 导入
- ⬜ 健康检查端点正常工作

## 🏗️ 轮次2：前端修正

### 目标

规范化 API 调用，持久化自选股列表。

### 步骤

#### 2.1 创建 API 模块

**文件**: `frontend/src/api/modules/quotes.ts`（新建）

```typescript
import http from '@/api/http'

export function fetchKline(symbol: string, period: string, count: number) { ... }
export function fetchSnapshot(symbol: string) { ... }
export function fetchSnapshotBatch(symbols: string[]) { ... }
```

#### 2.2 重构 RealtimeQuotes.vue — API 调用

**文件**: `frontend/src/views/RealtimeQuotes.vue`

替换硬编码的 `fetch` 调用为 `quotes.ts` 模块方法。

#### 2.3 自选股持久化

**文件**: `frontend/src/views/RealtimeQuotes.vue`

将 `watchlist` 从硬编码改为 `localStorage` 持久化。

#### 2.4 数据源显示修正

当前 `dataSource` 固定显示 `XtQuant`，应从 API 响应的 `data_source` 字段动态更新。

#### 2.5 动态刷新频率

当前固定 5 秒轮询，应根据市场状态动态调整：

| 时段 | 刷新间隔 | 说明 |
|------|---------|------|
| 集合竞价 / 开盘 | 3秒 | 高频 |
| 交易活跃时段 | 5秒 | 中频 |
| 午休 / 收盘后 / 周末 | 30秒 | 低频 |

实现方式：
1. 页面加载时调用 `GET /api/v1/quotes/market/status`
2. 使用返回的 `refresh_interval` 设置 `setInterval` 间隔
3. 定时重新获取市场状态（如每分钟），时段变化时自动调整

### 完成标准

- ⬜ 无硬编码 API 路径
- ⬜ 自选股列表刷新页面不丢失
- ⬜ 数据源显示与实际一致
- ⬜ 轮询间隔根据市场状态动态调整
- ⬜ 状态栏显示当前市场状态（交易中/午休/休市）

## 📊 数据表结构

### 1. 嵌套回测任务表 (nested_backtest_tasks)

存储嵌套回测任务的基本信息和配置。

```sql
CREATE TABLE nested_backtest_tasks (
    -- 主键
    task_id VARCHAR(64) PRIMARY KEY COMMENT '任务ID，格式: nested_backtest_YYYYMMDD_XXX',

    -- 任务基本信息
    task_name VARCHAR(255) NOT NULL COMMENT '任务名称',
    description TEXT COMMENT '任务描述',
    status ENUM('pending', 'running', 'completed', 'failed', 'stopped') NOT NULL DEFAULT 'pending' COMMENT '任务状态',
    priority TINYINT NOT NULL DEFAULT 5 COMMENT '优先级 1-10，10最高',

    -- 策略配置（JSON格式）
    strategy_levels JSON NOT NULL COMMENT '策略级别配置，包含所有级别的策略信息',
    execution_config JSON COMMENT '执行配置，包含交互、联合优化等设置',

    -- 回测配置
    backtest_config JSON NOT NULL COMMENT '回测配置，包含时间范围、账户、交易所等',

    -- 执行信息
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    started_at TIMESTAMP NULL COMMENT '开始执行时间',
    completed_at TIMESTAMP NULL COMMENT '完成时间',
    estimated_time INT COMMENT '预估耗时（秒）',
    actual_time INT COMMENT '实际耗时（秒）',

    -- 进度信息
    progress DECIMAL(5, 4) DEFAULT 0 COMMENT '执行进度 0-1',
    current_level INT COMMENT '当前执行的级别',
    current_date DATE COMMENT '当前处理的日期',

    -- 用户信息
    created_by VARCHAR(64) NOT NULL COMMENT '创建用户ID',

    -- 索引
    INDEX idx_status (status),
    INDEX idx_created_at (created_at),
    INDEX idx_created_by (created_by)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='嵌套回测任务表';
```

**strategy_levels字段结构**:
```json
{
  "levels": [
    {
      "level": 1,
      "name": "日频选股",
      "frequency": "day",
      "strategy": {
        "class": "TopkDropoutStrategy",
        "parameters": {
          "topk": 50,
          "n_drop": 5
        }
      },
      "information_extractor": {
        "class": "DailyFeatureExtractor",
        "config": {...}
      },
      "forecast_model": {
        "class": "lgb",
        "model_path": "/models/daily_model.pkl"
      },
      "decision_generator": {
        "class": "StockPickingGenerator",
        "config": {...}
      }
    }
  ]
}
```

### 2. 嵌套回测结果表 (nested_backtest_results)

存储嵌套回测任务的执行结果。

```sql
CREATE TABLE nested_backtest_results (
    -- 主键
    result_id BIGINT AUTO_INCREMENT PRIMARY KEY,
    task_id VARCHAR(64) NOT NULL COMMENT '任务ID',

    -- 整体性能指标
    return_rate DECIMAL(10, 6) COMMENT '收益率',
    annual_return DECIMAL(10, 6) COMMENT '年化收益率',
    sharpe_ratio DECIMAL(10, 6) COMMENT '夏普比率',
    max_drawdown DECIMAL(10, 6) COMMENT '最大回撤',
    calmar_ratio DECIMAL(10, 6) COMMENT '卡尔马比率',
    turnover_rate DECIMAL(10, 6) COMMENT '换手率',

    -- 各级别性能贡献
    level_performance JSON COMMENT '各级别性能贡献',
    interaction_analysis JSON COMMENT '策略交互分析',
    performance_comparison JSON COMMENT '性能对比分析',

    -- 详细指标（文件路径）
    returns_file_path VARCHAR(512) COMMENT '日收益率文件路径',
    positions_file_path VARCHAR(512) COMMENT '持仓记录文件路径',
    trades_file_path VARCHAR(512) COMMENT '交易记录文件路径',
    metrics_file_path VARCHAR(512) COMMENT '详细指标文件路径',

    -- 时间戳
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',

    -- 外键约束
    FOREIGN KEY (task_id) REFERENCES nested_backtest_tasks(task_id) ON DELETE CASCADE,

    -- 索引
    INDEX idx_task_id (task_id),
    INDEX idx_return_rate (return_rate),
    INDEX idx_sharpe_ratio (sharpe_ratio)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='嵌套回测结果表';
```

### 3. 策略交互分析表 (strategy_interaction_analysis)

存储策略交互分析的详细结果。

```sql
CREATE TABLE strategy_interaction_analysis (
    -- 主键
    analysis_id BIGINT AUTO_INCREMENT PRIMARY KEY,

    -- 关联信息
    task_id VARCHAR(64) COMMENT '关联的回测任务ID',
    comparison_id VARCHAR(64) COMMENT '关联的对比任务ID',

    -- 分析配置
    analysis_config JSON COMMENT '分析配置，包含指标、敏感性分析等',

    -- 交互效应分析
    interaction_effects JSON NOT NULL COMMENT '交互效应分析结果',
    sensitivity_analysis JSON COMMENT '敏感性分析结果',
    ablation_study JSON COMMENT '消融实验结果',

    -- 推荐建议
    recommendations JSON COMMENT '优化建议',

    -- 时间戳
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',

    -- 外键约束
    FOREIGN KEY (task_id) REFERENCES nested_backtest_tasks(task_id) ON DELETE SET NULL,

    -- 索引
    INDEX idx_task_id (task_id),
    INDEX idx_comparison_id (comparison_id),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='策略交互分析表';
```

### 5. 嵌套策略执行日志表 (nested_strategy_execution_logs)

存储嵌套策略执行的详细日志。

```sql
CREATE TABLE nested_strategy_execution_logs (
    -- 主键
    log_id BIGINT AUTO_INCREMENT PRIMARY KEY,

    -- 关联信息
    task_id VARCHAR(64) NOT NULL COMMENT '任务ID',
    level INT NOT NULL COMMENT '策略级别',

    -- 执行信息
    trade_date DATE NOT NULL COMMENT '交易日期',
    timestamp TIMESTAMP NOT NULL COMMENT '执行时间戳',

    -- 输入
    input_data JSON COMMENT '输入数据',

    -- 决策
    decision JSON COMMENT '决策结果',

    -- 输出
    output_data JSON COMMENT '输出数据',

    -- 性能指标
    execution_time_ms INT COMMENT '执行耗时（毫秒）',
    memory_usage_mb DECIMAL(10, 2) COMMENT '内存使用（MB）',

    -- 时间戳
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',

    -- 外键约束
    FOREIGN KEY (task_id) REFERENCES nested_backtest_tasks(task_id) ON DELETE CASCADE,

    -- 索引
    INDEX idx_task_id (task_id),
    INDEX idx_level (level),
    INDEX idx_trade_date (trade_date),
    INDEX idx_timestamp (timestamp)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='嵌套策略执行日志表';
```

### 6. 策略级别配置表 (strategy_level_configs)

存储可复用的策略级别配置模板。

```sql
CREATE TABLE strategy_level_configs (
    -- 主键
    config_id BIGINT AUTO_INCREMENT PRIMARY KEY,

    -- 配置信息
    config_name VARCHAR(255) NOT NULL COMMENT '配置名称',
    level INT NOT NULL COMMENT '策略级别',
    description TEXT COMMENT '配置描述',

    -- 策略配置

    tags JSON COMMENT '标签，用于分类和搜索',

    -- 用户信息
    created_by VARCHAR(64) NOT NULL COMMENT '创建用户ID',
    is_public BOOLEAN DEFAULT FALSE COMMENT '是否公开',

    -- 时间戳
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',

    -- 索引
    UNIQUE KEY uk_config_name_level (config_name, level),
    INDEX idx_level (level),
    INDEX idx_tags (tags(100)),
    INDEX idx_is_public (is_public)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='策略级别配置表';
```

## 🗂️ 索引设计

### 主要索引

**nested_backtest_tasks表**:
- PRIMARY KEY: `task_id`
- INDEX `idx_status`: 支持按状态查询任务
- INDEX `idx_created_at`: 支持按时间范围查询
- INDEX `idx_created_by`: 支持按用户查询

**nested_backtest_results表**:
- PRIMARY KEY: `result_id`
- INDEX `idx_task_id`: 支持按任务查询结果
- INDEX `idx_return_rate`: 支持按收益率排序
- INDEX `idx_sharpe_ratio`: 支持按夏普比率排序

**strategy_interaction_analysis表**:
- PRIMARY KEY: `analysis_id`
- INDEX `idx_task_id`: 支持按任务查询分析结果
- INDEX `idx_comparison_id`: 支持按对比任务查询
- INDEX `idx_created_at`: 支持按时间范围查询

**nested_strategy_execution_logs表**:
- PRIMARY KEY: `log_id`
- INDEX `idx_task_id`: 支持按任务查询日志
- INDEX `idx_level`: 支持按级别查询
- INDEX `idx_trade_date`: 支持按日期查询
- INDEX `idx_timestamp`: 支持按时间戳查询
- COMPOSITE INDEX `(task_id, level, trade_date)`: 支持复合查询

## 💾 数据存储策略

### 大文件存储

对于以下大数据量字段，建议使用文件存储而非数据库存储：

1. **returns_file_path**: 日收益率数据
   - 存储格式: CSV或Parquet
   - 存储位置: `/data/nested_backtest/{task_id}/returns.csv`

2. **positions_file_path**: 持仓记录
   - 存储格式: CSV或Parquet
   - 存储位置: `/data/nested_backtest/{task_id}/positions.csv`

3. **trades_file_path**: 交易记录
   - 存储格式: CSV或Parquet
   - 存储位置: `/data/nested_backtest/{task_id}/trades.csv`

4. **metrics_file_path**: 详细指标
   - 存储格式: JSON或HDF5
   - 存储位置: `/data/nested_backtest/{task_id}/metrics.json`

### 文件命名规范


## 🔄 数据生命周期

### 数据保留策略

**任务数据**:
- 运行中任务: 永久保留
- 已完成任务: 保留180天
- 失败任务: 保留30天
- 已停止任务: 保留90天

**结果数据**:
- 核心指标: 永久保留（存储在数据库）
- 详细数据文件: 保留180天
- 日志数据: 保留90天

**清理策略**:
- 每周清理一次过期数据
- 清理前发送通知给用户
- 重要任务可以标记为"永久保留"

## ⚠️ 数据完整性约束

### 外键约束

1. `nested_backtest_results.task_id` → `nested_backtest_tasks.task_id`
   - ON DELETE CASCADE: 删除任务时同时删除结果

2. `strategy_interaction_analysis.task_id` → `nested_backtest_tasks.task_id`
   - ON DELETE SET NULL: 删除任务时保留分析结果

3. `nested_strategy_execution_logs.task_id` → `nested_backtest_tasks.task_id`
   - ON DELETE CASCADE: 删除任务时同时删除日志

### 检查约束

建议在应用层实现以下检查：

1. **任务状态转换**:
   - pending → running
   - running → completed/failed/stopped
   - 不允许逆转换

2. **进度值**:
   - progress 必须在 [0, 1] 范围内

3. **收益率范围**:
   - return_rate 应在合理范围内（如 -1 到 10）

4. **级别一致性**:
   - level 必须在 strategy_levels 定义的范围内

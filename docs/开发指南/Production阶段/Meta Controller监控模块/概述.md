## 🎯 模块定位

### 核心价值
**实时监控ML模型性能，自动检测模型衰减，实现智能模型切换**

> **关键洞察**: ML模型上线后会逐渐衰减（市场环境变化、数据分布偏移），需要持续监控和自动切换机制

## 📊 核心功能详解

### 1. 实时IC/IR跟踪 (P1核心)

**功能描述**:
- 每分钟计算模型预测的IC（信息系数）
- 跟踪IR（信息比率）
- 生成IC时间序列趋势图

**计算方式**:
```python
IC = correlation(预测评分, 实际收益)  # Spearman相关系数
IR = IC的均值 / IC的标准差
IC_MA7 = 过去7天IC移动平均
```

**告警阈值**:
| 指标 | 正常范围 | 警告 | 严重 |
|------|---------|------|------|
| IC | > 0.03 | 0.02-0.03 | < 0.02 |
| IC衰减 | < 30% | 30-60% | > 60% |
| IR | > 0.5 | 0.3-0.5 | < 0.3 |

### 2. 模型衰减检测 (P1核心)

**功能描述**:
- 监控IC的滑动窗口趋势
- 检测预测分布偏移
- 自动识别模型失效

**衰减信号**:
1. **IC下降信号**: IC_MA7 < IC_MA30 * 0.7
2. **分布偏移信号**: 预测分布KS检验 > 0.2
3. **准确率下降**: 5日准确率 < 70% * 30日准确率

### 3. 自动模型切换 (P1核心)

**功能描述**:
- 检测到衰减时自动切换到备用模型
- 支持多级备用模型配置
- 切换过程不中断服务

**切换策略**:
```python
if IC < 0.02 或 衰减 > 60%:
    切换到备用模型 v1.0 (最近稳定版本)
elif IC < 0.03 或 衰减 > 40%:
    发送警告通知
    准备备用模型待命
```

### 4. 性能指标监控 (P1核心)

**监控指标**:
| 指标 | 说明 | 告警阈值 |
|------|------|----------|
| 预测延迟 | 每次预测的P95延迟 | > 100ms |
| 预测准确率 | 滚动5日准确率 | < 70% |
| 预测分布 | 预测评分分布偏移 | > 20% |
| 模型可用性 | 模型服务可用率 | < 99.9% |

## 🏗️ 技术架构

### 核心技术
- **后端**: Python + FastAPI
- **服务路径**: `services/production/meta_controller_service.py`
- **API路径**: `api/v1/production/meta_router.py`


## 📊 数据表结构

### 1. 嵌套执行实例表 (nested_execution_instances)

存储嵌套执行系统的实例信息和运行状态。

```sql
CREATE TABLE nested_execution_instances (
    -- 主键
    execution_id VARCHAR(64) PRIMARY KEY COMMENT '执行实例ID，格式: nested_exec_YYYYMMDD_XXX',

    -- 实例基本信息
    execution_name VARCHAR(255) NOT NULL COMMENT '执行实例名称',
    description TEXT COMMENT '执行实例描述',
    status ENUM('starting', 'running', 'stopping', 'stopped', 'error') NOT NULL DEFAULT 'starting' COMMENT '运行状态',

    -- 嵌套配置（JSON格式）
    nested_config JSON NOT NULL COMMENT '嵌套策略配置，包含所有级别',
    execution_mode ENUM('realtime', 'simulation') NOT NULL DEFAULT 'realtime' COMMENT '执行模式',
    execution_config JSON COMMENT '执行配置，包含交互、联合优化等',

    -- 风控配置
    risk_control JSON COMMENT '风控配置',

    -- 运行信息
    started_at TIMESTAMP NOT NULL COMMENT '启动时间',
    stopped_at TIMESTAMP NULL COMMENT '停止时间',
    uptime_hours DECIMAL(10, 2) COMMENT '运行时长（小时）',
    restart_count INT DEFAULT 0 COMMENT '重启次数',

    -- 用户信息
    created_by VARCHAR(64) NOT NULL COMMENT '创建用户ID',

    -- 索引
    INDEX idx_status (status),
    INDEX idx_started_at (started_at),
    INDEX idx_created_by (created_by)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='嵌套执行实例表';
```

**nested_config字段结构**:
```json
{
  "level1": {
    "name": "日频选股",

    "execute_time": "09:00",

      "parameters": {"topk": 50, "n_drop": 5}
    },
    "information_extractor": {...},
    "forecast_model": {...},
    "decision_generator": {...}
  },
  "level2": {
    "name": "日内择时",
    "frequency": "30min",
    "execute_time": "09:30-15:00",
    "strategy": {...}
  },
  "level3": {
    "name": "订单拆分",
    "frequency": "1min",
    "execute_time": "09:30-15:00",
    "strategy": {...}
```

### 2. 嵌套执行级别状态表 (nested_execution_level_status)

存储各级子策略的实时执行状态。

```sql
CREATE TABLE nested_execution_level_status (
    -- 主键
    status_id BIGINT AUTO_INCREMENT PRIMARY KEY,

    -- 关联信息
    execution_id VARCHAR(64) NOT NULL COMMENT '执行实例ID',
    level INT NOT NULL COMMENT '策略级别',

    -- 基本信息
    level_name VARCHAR(255) NOT NULL COMMENT '级别名称',
    status ENUM('idle', 'running', 'error', 'disabled') NOT NULL DEFAULT 'idle' COMMENT '级别状态',

    -- 上次执行信息
    last_execution_at TIMESTAMP NULL COMMENT '上次执行时间',
    last_execution_duration_ms INT COMMENT '上次执行耗时（毫秒）',
    last_execution_result JSON COMMENT '上次执行结果',
    last_execution_error TEXT COMMENT '上次执行错误',

    -- 下次执行信息
    next_execution_at TIMESTAMP NULL COMMENT '下次执行时间',

    -- 模型状态
    model_status JSON COMMENT '模型状态信息',

    -- 统计信息
    total_executions INT DEFAULT 0 COMMENT '总执行次数',
    successful_executions INT DEFAULT 0 COMMENT '成功执行次数',
    failed_executions INT DEFAULT 0 COMMENT '失败执行次数',

    -- 时间戳
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',

    -- 外键约束
    FOREIGN KEY (execution_id) REFERENCES nested_execution_instances(execution_id) ON DELETE CASCADE,

    -- 索引
    UNIQUE KEY uk_execution_id_level (execution_id, level),
    INDEX idx_status (status),
    INDEX idx_next_execution_at (next_execution_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='嵌套执行级别状态表';
```

**model_status字段结构**:
```json
{
  "model_version": "v1.2.0",
  "last_updated": "2024-02-09",
  "prediction_accuracy": 0.62,
  "model_path": "/models/daily_model.pkl",
  "online_serving_enabled": true,
  "last_online_update": "2024-02-09"
}
```

### 3. 嵌套执行订单表 (nested_execution_orders)

存储嵌套执行系统生成的所有订单。

```sql
CREATE TABLE nested_execution_orders (
    -- 主键
    order_id VARCHAR(64) PRIMARY KEY COMMENT '订单ID，格式: order_YYYYMMDD_XXX',

    -- 关联信息
    execution_id VARCHAR(64) NOT NULL COMMENT '执行实例ID',
    level INT NOT NULL COMMENT '策略级别',
    parent_order_id VARCHAR(64) COMMENT '父订单ID（用于拆单）',

    -- 订单基本信息
    stock VARCHAR(16) NOT NULL COMMENT '股票代码',
    side ENUM('buy', 'sell') NOT NULL COMMENT '买卖方向',
    order_type ENUM('limit', 'market', 'stop') NOT NULL DEFAULT 'limit' COMMENT '订单类型',

    -- 订单数量和价格
    total_quantity INT NOT NULL COMMENT '总数量',
    executed_quantity INT DEFAULT 0 COMMENT '已成交数量',
    remaining_quantity INT NOT NULL COMMENT '剩余数量',

    limit_price DECIMAL(10, 2) COMMENT '限价',
    avg_price DECIMAL(10, 2) COMMENT '平均成交价格',
    target_price DECIMAL(10, 2) COMMENT '目标价格',

    -- 执行信息
    execution_style VARCHAR(32) COMMENT '执行方式（twap/vwap/rl）',
    execution_progress DECIMAL(5, 4) DEFAULT 0 COMMENT '执行进度 0-1',
    slippage DECIMAL(10, 6) COMMENT '滑点',

    -- 订单状态
    status ENUM('pending', 'partial', 'completed', 'failed', 'cancelled') NOT NULL DEFAULT 'pending' COMMENT '订单状态',

    -- 时间信息
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    updated_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    completed_at TIMESTAMP NULL COMMENT '完成时间',

    -- 额外信息
    metadata JSON COMMENT '额外元数据',

    -- 外键约束
    FOREIGN KEY (execution_id) REFERENCES nested_execution_instances(execution_id) ON DELETE CASCADE,
    FOREIGN KEY (parent_order_id) REFERENCES nested_execution_orders(order_id) ON DELETE SET NULL,

    -- 索引
    INDEX idx_execution_id (execution_id),
    INDEX idx_level (level),
    INDEX idx_stock (stock),
    INDEX idx_status (status),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='嵌套执行订单表';
```

### 4. 策略调整记录表 (strategy_adjustment_logs)

记录所有子策略的动态调整历史。

```sql
CREATE TABLE strategy_adjustment_logs (
    -- 主键
    adjustment_id BIGINT AUTO_INCREMENT PRIMARY KEY,

    -- 关联信息
    execution_id VARCHAR(64) NOT NULL COMMENT '执行实例ID',
    level INT NOT NULL COMMENT '策略级别',

    -- 调整信息
    parameter VARCHAR(64) NOT NULL COMMENT '调整的参数名',
    previous_value JSON COMMENT '调整前的值',
    new_value JSON NOT NULL COMMENT '调整后的值',

    -- 调整原因和类型
    reason TEXT COMMENT '调整原因',
    adjustment_type ENUM('manual', 'auto', 'emergency') NOT NULL COMMENT '调整类型',
    trigger_condition JSON COMMENT '触发条件（自动调整）',

    -- 调整配置
    effect_time ENUM('immediate', 'scheduled') NOT NULL DEFAULT 'immediate' COMMENT '生效时间',
    duration ENUM('permanent', 'temporary') NOT NULL COMMENT '持续时间',
    expiry_time TIMESTAMP NULL COMMENT '过期时间（临时调整）',

    -- 预期影响
    expected_impact TEXT COMMENT '预期影响说明',

    -- 实际影响（事后统计）
    actual_impact JSON COMMENT '实际影响统计',

    -- 时间戳
    created_at TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    applied_at TIMESTAMP NULL COMMENT '生效时间',

    -- 用户信息
    created_by VARCHAR(64) COMMENT '创建用户ID（手动调整）',

    -- 外键约束
    FOREIGN KEY (execution_id) REFERENCES nested_execution_instances(execution_id) ON DELETE CASCADE,

    -- 索引
    INDEX idx_execution_id (execution_id),
    INDEX idx_level (level),
    INDEX idx_created_at (created_at)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='策略调整记录表';
```

## 🗂️ 索引设计

### 主要索引

**nested_execution_instances表**:
- PRIMARY KEY: `execution_id`
- INDEX `idx_status`: 支持按状态查询实例
- INDEX `idx_started_at`: 支持按时间范围查询
- INDEX `idx_created_by`: 支持按用户查询

**nested_execution_level_status表**:
- PRIMARY KEY: `status_id`
- UNIQUE INDEX `uk_execution_id_level`: 确保每个实例的每个级别只有一条记录
- INDEX `idx_status`: 支持按状态查询
- INDEX `idx_next_execution_at`: 支持查询待执行的级别

**nested_execution_orders表**:
- PRIMARY KEY: `order_id`
- INDEX `idx_execution_id`: 支持按实例查询订单
- INDEX `idx_level`: 支持按级别查询订单

## 💾 数据存储策略

### 热数据与冷数据分离

**热数据**（频繁访问）:
- `nested_execution_level_status`: 存储在MySQL，频繁读写
- `realtime_order_monitor`: 存储在MySQL，实时更新
- `nested_execution_logs`: 最近7天存储在MySQL，超过7天归档

**冷数据**（历史记录）:
- `strategy_adjustment_logs`: 永久保留在MySQL
- `strategy_switch_logs`: 永久保留在MySQL
- `nested_execution_performance`: 保留180天，超过归档到对象存储
- `nested_execution_orders`: 已完成订单保留180天，超过归档

### 归档策略

**归档规则**:
1. **日志归档**: 超过7天的日志归档到对象存储（如S3）
2. **订单归档**: 已完成且超过180天的订单归档
3. **性能数据归档**: 超过180天的性能数据归档
4. **调整记录**: 永久保留，不归档

**归档格式**:
```
s3://myquant-archives/nested_execution/
├── logs/
│   └── yyyy/mm/
│       └── nested_execution_logs_YYYYMMDD.parquet
├── orders/
│   └── yyyy/mm/
│       └── nested_execution_orders_YYYYMMDD.parquet
└── performance/
    └── yyyy/mm/
        └── nested_execution_performance_YYYYMMDD.parquet
```

## ⚠️ 数据完整性约束

### 外键约束

1. `nested_execution_level_status.execution_id` → `nested_execution_instances.execution_id`
   - ON DELETE CASCADE: 删除实例时同时删除级别状态

2. `nested_execution_orders.execution_id` → `nested_execution_instances.execution_id`
   - ON DELETE CASCADE: 删除实例时同时删除订单

3. `nested_execution_orders.parent_order_id` → `nested_execution_orders.order_id`
   - ON DELETE SET NULL: 删除父订单时保留子订单

4. `realtime_order_monitor.order_id` → `nested_execution_orders.order_id`
   - ON DELETE CASCADE: 删除订单时同时删除监控记录

### 检查约束

建议在应用层实现以下检查：

1. **订单数量一致性**:
   ```python
   assert order.total_quantity == order.executed_quantity + order.remaining_quantity
   ```

2. **执行进度范围**:
   ```python
   assert 0 <= order.execution_progress <= 1
   ```

3. **级别顺序**:
   ```python
   # Level 1必须是最高的级别
   assert any(level['level'] == 1 for level in nested_config['levels'])
   ```

4. **执行时间合理性**:
   ```python
   # 日内策略的执行时间必须在交易时间内
   if level['frequency'] in ['30min', '5min', '1min']:
       assert '09:30' <= level['execute_time'] <= '15:00'
   ```

## 🎯 模块定位

### 在三阶段中的位置
- **Research阶段**: 负责机器学习模型的训练、预测和评估
- **核心价值**: 为量化策略提供基于ML的预测能力

### 与QLib高级模块的集成

本模块深度集成QLib高级模块，实现自动化ML训练流水线：

| QLib高级模块 | 集成方式 | 作用 |
|-------------|---------|------|
| **Workflow管理模块** | Task → Dataset → Model → Recorder | 自动化训练流程编排，追踪实验状态 |
| **Task任务管理模块** | TaskGen → TaskManager → Trainer → Collector | 批量任务生成、并行训练、结果收集 |

### 与其他模块的关系
- **上游**:
  - **数据管理模块** - 提供清洗后的K线数据
  - **因子计算模块** - 提供150+技术指标作为特征
  - **因子分析模块** - 提供因子统计信息
  - **因子评估模块** - 提供因子有效性评估结果

- **下游**:
  - **Production阶段** - 使用训练好的模型进行实时预测
  - **Production阶段** - 基于ML预测信号进行交易决策
  - **回测模块** - 评估ML策略的历史表现

## 📊 核心功能

### 1. 模型训练
支持多种机器学习算法的模型训练：

#### 传统机器学习模型
- **LightGBM** (`LGBModel`)
  - 优点：训练快速、内存效率高、支持GPU
  - 适用场景：表格数据、因子预测
  - 超参数：num_leaves, learning_rate, n_estimators

- **XGBoost** (`XGBModel`)
  - 优点：性能优异、鲁棒性强
  - 适用场景：分类、回归任务
  - 超参数：max_depth, eta, subsample

- **Random Forest** (`RFModel`)
  - 优点：可解释性强、抗过拟合
  - 适用场景：特征重要性分析
  - 超参数：n_estimators, max_depth

- **Linear Model** (`LinearModel`)
  - 优点：简单快速、基线模型
  - 适用场景：快速原型
  - 超参数：正则化参数

#### 深度学习模型
- **LSTM** (`LSTM`)
  - 优点：捕捉时序依赖、记忆长期模式
  - 适用场景：价格预测、时序分类
  - 超参数：hidden_size, num_layers, dropout

- **GRU** (`GRU`)
  - 优点：比LSTM更快、参数更少
  - 适用场景：时序预测
  - 超参数：hidden_size, num_layers

- **Transformer** (`Transformer`)
  - 优点：并行计算、注意力机制
  - 适用场景：多变量时序预测
  - 超参数：d_model, nhead, num_layers

- **MLP** (`MLPModel`)
  - 优点：通用函数逼近器
  - 适用场景：非线性映射
  - 超参数：hidden_sizes, activation

### 2. 模型预测
使用训练好的模型进行预测：
- **单股票预测**: 对单个股票进行预测
- **批量预测**: 对多股票池进行批量预测
- **实时预测**: 在Production阶段实时预测

### 3. 模型评估

评估模型性能的多种指标：

#### 分类任务 (用于预测涨跌)
- **准确率 (Accuracy)**: 预测正确的比例
- **精确率 (Precision)**: 预测上涨中实际上涨的比例
- **召回率 (Recall)**: 实际上涨中被正确预测到的比例
- **F1分数 (F1-Score)**: 精确率和召回率的调和平均
- **AUC-ROC**: 分类器区分涨和跌的能力（0.5=随机，1.0=完美）

> **谁用**: 研究员判断模型能否区分涨跌
> **参考值**: AUC > 0.6 有一定预测能力，AUC > 0.7 较好，AUC > 0.8 很好

#### 回归任务 (用于预测收益率)
- **均方误差 (MSE)**: 预测误差的平方平均
- **平均绝对误差 (MAE)**: 预测误差的绝对值平均
- **R²分数 (R² Score)**: 模型解释数据变化的程度
- **IC (Information Coefficient)**: 预测值与实际收益的皮尔逊相关系数
- **RankIC**: 预测值与实际收益的斯皮尔曼等级相关系数

> **谁用**: 研究员判断模型预测收益的能力
> **参考值**: IC > 0.03 有预测能力，IC > 0.05 较好，IC > 0.08 很好
> **RankIC** 更稳健，不受极端值影响

### 4. 超参数优化

自动化超参数搜索，让模型效果更好：

- **网格搜索** (Grid Search): 遍历所有参数组合
  - 优点: 保证找到最优解
  - 缺点: 参数多时计算量大
  - 适用: 参数空间较小的情况

- **随机搜索** (Random Search): 随机采样参数空间
  - 优点: 比网格搜索快
  - 缺点: 可能错过最优解
  - 适用: 参数空间较大的情况

- **贝叶斯优化** (Bayesian Optimization): 基于历史结果智能搜索
  - 优点: 效率高，少量尝试就能找到好参数
  - 缺点: 实现复杂
  - 适用: 大部分情况（推荐）

- **Optuna**: 高级超参数优化框架
  - 优点: 支持贝叶斯优化、智能剪枝、并行搜索
  - 适用: 生产级超参数优化

> **谁用**: 研究员不想手动一个个试参数，让算法自动找最优参数组合
> **例子**: 原来要手动试50种 `num_leaves` + `learning_rate` 组合，现在自动找到最优的

### 5. 特征工程

提升模型效果的特征处理：

- **特征选择**: ✅ 自动选择最重要的特征
  - 目的: 去掉噪音特征，提升模型泛化能力
  - 方法: 基于重要性、基于相关性
  - 适用: 特征太多（150+因子）时
  - API: `/ml/features/select`

- **特征重要性**: ✅ 分析各特征贡献度
  - 目的: 理解哪些因子对预测最有帮助
  - 输出: 特征重要性排名
  - 适用: 因子分析、模型解释性
  - API: `/ml/features`

- **特征变换**: ✅ 标准化、归一化、鲁棒化
  - 目的: 让不同量纲的特征可比，加速模型训练
  - 方法:
    - 标准化 (StandardScaler): 转为均值0方差1
    - 归一化 (MinMaxScaler): 转为0-1范围
    - 鲁棒化 (RobustScaler): 中位数/IQR，对异常值鲁棒
    - 对数变换: 处理偏态分布
  - 适用: 大部分模型都需要
  - API: `/ml/features/transform`

- **特征去冗余**: ✅ 去除高度相关特征
  - 目的: 避免多重共线性问题
  - 方法: 计算特征间相关矩阵，移除高相关特征
  - API: `/ml/features/remove-correlated`

- **特征组合**: ✅ 创建多项式特征
  - 目的: 捕捉特征间的非线性关系
  - 方法: 特征相乘、特征比值、多项式特征
  - 适用: 简单模型捕捉复杂模式
  - API: `/ml/features/polynomial`

> **谁用**: 研究员处理150+技术指标时，不知道用哪些、怎么用
> **例子**: 从50个因子选出10个最有效的，或者把原始因子做标准化

## 🎯 应用场景

### 1. 股票涨跌预测（分类）
- **任务类型**: 二分类（涨/跌）
- **特征**: 技术指标、基本面数据
- **标签**: 未来N天收益率 > 0
- **模型**: XGBoost、LightGBM、LSTM
- **评估**: 准确率、AUC、F1

### 2. 收益率预测（回归）
- **任务类型**: 回归
- **特征**: 技术指标、历史收益率
- **标签**: 未来N天收益率
- **模型**: LightGBM、LSTM、GRU
- **评估**: MSE、MAE、IC

### 3. 因子权重学习
- **任务类型**: 优化
- **特征**: 多个因子
- **目标**: 最大化组合夏普比率
- **模型**: 线性回归、强化学习
- **评估**: 回测收益、风险指标

---

### 4. 多因子时序建模（LSTM/GRU深度学习应用）⭐

#### 任务类型
- **时序分类**: 基于历史序列预测涨跌
- **时序回归**: 基于历史序列预测收益率

#### LSTM应用
- **场景**: 50个技术因子的20天时序预测
- **输入形状**: `(batch_size, 20, 50)` - 20天历史，50个因子
- **核心优势**: 捕捉因子之间的时序依赖关系
  - MA5如何影响MA20（均线交叉关系）
  - RSI如何影响MACD（指标共振）
  - 过去10天的波动率如何影响今天
- **架构**: LSTM(128) → Dropout(0.3) → LSTM(64) → Dense(1)
- **输出**: 涨跌概率 + 时序注意力权重
- **评估**: 时序准确率、时序IC

#### GRU应用
- **场景**: 快速时序建模（比LSTM更快）
- **优势**: 参数更少，训练更快，适合实时更新
- **架构**: GRU(128) → Dropout(0.3) → Dense(1)
- **应用**: 在线学习场景（新数据快速适应）
- **评估**: 预测延迟、在线学习效果

---

### 5. 板块轮动与跨股票注意力（Transformer深度学习应用）⭐

#### 任务类型
- **板块轮动预测**: 哪个板块会上涨
- **跨股票相关性**: 发现股票间的隐藏关联
- **市场情绪传播**: 恐慌/贪婪如何在股票间扩散

#### Transformer应用
- **场景**: 4000只股票的全局注意力建模
- **输入形状**: `(batch_size, 4000, 10, 50)` - 4000股票，10天历史，50因子
- **核心优势**:
  - **自注意力机制**: 每只股票"观察"其他所有股票
  - **位置编码**: 区分股票身份
  - **多头注意力**: 同时捕捉多种市场模式
- **架构**:
  ```
  Transformer Encoder(d_model=256, nhead=8, num_layers=6)
  → Cross-Attention → Prediction
  ```
- **应用**:
  - 预测明天哪些板块会上涨
  - 发现板块轮动模式（科技→消费→金融）
  - 识别龙头股及其跟随股票
- **评估**: 板块分类准确率、轮动策略收益

---

### 6. 非线性因子组合与深度特征提取（MLP深度学习应用）⭐

#### 任务类型
- **深度特征提取**: 从50个因子中提取高阶特征
- **非线性组合优化**: 学习因子间的非线性关系

#### MLP应用
- **场景**: 替代传统线性因子加权
- **架构**:
  ```
  Input(50)
  → Dense(256, ReLU, Dropout(0.3))
  → Dense(128, ReLU, Dropout(0.3))
  → Dense(64, ReLU, Dropout(0.3))
  → Dense(1, Sigmoid)  # 输出0-1买入评分
  ```
- **核心优势**:
  - **自动特征工程**: 隐藏层自动学习因子交互
  - **非线性组合**: 可以学习因子1*因子2的复杂关系
  - **端到端学习**: 从因子直接到买入评分
- **应用对比**:
  ```
  传统: Score = 0.3*MA5 + 0.2*RSI + 0.5*MACD
  MLP:   Score = MLP([MA5, RSI, MACD, ...])  # 自动学习最优组合
  ```
- **输出**: 0-1之间的连续买入评分（比分类更细粒度）
- **评估**: 买入评分区分度、回测收益对比

### 7. 异常检测与市场状态识别（无监督/半监督深度学习）⭐

#### 任务类型
- **市场状态识别**: 识别牛市/熊市/震荡市
- **异常检测**: 发现市场异常波动

#### LSTM/GRU Autoencoder应用
- **场景**: 无监督学习市场模式
- **方法**: LSTM-Autoencoder重构市场数据
- **架构**:
  ```
  LSTM Encoder(64) → Bottleneck(16) → LSTM Decoder(64)
  ```
- **输入**: 市场指数、多股票收益率
- **应用**:
  - 重构误差 = 实际值 - 重构值
  - 当重构误差 > 阈值 → 市场异常
  - 降维到低维空间 → 市场状态识别
- **输出**:
  - 市场状态标签（牛市/熊市/震荡）
  - 异常分数（市场是否异常波动）
- **评估**: 市场状态识别准确率、异常检测召回率

### 深度学习模型应用总结

| 模型 | 核心优势 | 主要应用 | 数据需求 | 计算需求 |
|------|---------|---------|---------|---------|
| **LSTM** | 时序依赖 | 多因子时序预测（20天历史） | 中等 | 高 |
| **GRU** | 快速训练 | 在线学习、实时预测 | 中等 | 中等 |
| **Transformer** | 全局注意力 | 板块轮动、跨股票相关性 | 大 | 高 |
| **MLP** | 非线性组合 | 深度特征提取、买入评分生成 | 小 | 低 |
| **Autoencoder** | 模式识别 | 市场状态识别、异常检测 | 中等 | 高 |

**应用原则**:
- ✅ **LSTM/GRU**: 时序建模主力（20%应用场景）
- ✅ **Transformer**: 特殊场景（板块轮动、全局模式）
- ✅ **MLP**: 组合优化（10%应用场景）
- ✅ **Autoencoder**: 辅助功能（5%应用场景）
- ✅ **XGBoost/LightGBM**: 表格数据主力（70%应用场景）

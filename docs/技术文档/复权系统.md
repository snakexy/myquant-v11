## 2. 核心模块详解

### 2.1 AdjustmentFactorService（复权因子服务）

**文件**: `backend/src/myquant/core/market/services/adjustment_factor_service.py`

#### 职责
- 管理复权因子表的计算和缓存
- 支持两种复权方式的因子计算
- 提供快速查表应用功能

#### 关键方法

| 方法 | 说明 | 性能 |
|------|------|------|
| `get_factor_table(symbol, adjust_type)` | 获取复权因子表（带两级缓存） | 首次 200-500ms，后续 <5ms |
| `apply_factors(df, factor_table)` | 应用复权因子到K线数据 | O(n) 查表，<5ms/1000条 |
| `calculate_front_factors(xdxr_data)` | 计算前复权累积因子 | 约 100-300ms |
| `calculate_front_ratio_factors(xdxr_data)` | 计算等比复权独立因子 | 约 50-100ms |

#### 缓存层级

```python
# L1 内存缓存
self._factor_cache = TTLCache(maxsize=500, ttl=7 * 24 * 3600)

# L2 文件缓存（Binary Pickle，比 JSON 快 5-10 倍）
data/xdxr_cache/{symbol}/factors_{adjust_type}.pkl
```

#### 使用示例

```python
from myquant.core.market.services.adjustment_factor_service import get_adjustment_factor_service

# 获取服务实例
service = get_adjustment_factor_service()

# 获取因子表（自动使用缓存）
factor_table = service.get_factor_table('600519.SH', 'front')

# 应用到K线数据
df_adjusted = service.apply_factors(df_raw, factor_table)
```

### 2.2 XDXR 数据缓存（高性能读取）

**文件**: `backend/src/myquant/core/market/services/seamless_service.py`

#### 优化点

| 优化项 | 实现方式 | 性能提升 |
|--------|---------|---------|
| 二进制格式 | Pickle 替代 JSON | 5-10x |
| 目录创建优化 | 类级别标记，只检查一次 | 减少 IO |
| 文件检查优化 | 合并 exists + stat 调用 | 减少 IO |
| 格式自动回退 | .pkl 不存在则尝试 .json | 兼容旧数据 |

#### 存储结构

```
data/xdxr_cache/
├── 000001_SZ/
│   ├── 000001_SZ.pkl          # 二进制格式（优先）
│   ├── 000001_SZ.json         # JSON格式（兼容）
│   ├── factors_front.pkl      # 前复权因子表
│   └── factors_front_ratio.pkl # 等比复权因子表
```

#### 文件读取性能对比

| 格式 | 文件大小 | 读取时间 | 适用场景 |
|------|---------|---------|---------|
| JSON | 100KB | 10-20ms | 调试、兼容 |
| Pickle | 65KB | 1-3ms | 生产环境（推荐） |

### 2.1 混合模式复权（已废弃）

**废弃原因**: 通达信使用统一前复权算法，所有周期使用相同逻辑

**之前策略**（已废弃）:
- 日线(1d): 累积因子
- 分钟线: 等比复权

**当前策略**（通达信标准）:
- 所有周期统一使用前复权（不累积，只用最新除权日因子）

## 3. 回测场景最佳实践

### 3.1 回测复权流程

```python
# 1. 定义回测股票池
stock_pool = ['000001.SZ', '600519.SH', ...]  # 几十~几百只

# 2. 批量预热（回测开始前一次性执行）
from myquant.core.market.services.watchlist_prewarmer import WatchlistPrewarmer

prewarmer = WatchlistPrewarmer()
prewarmer.set_watchlist(stock_pool)
prewarmer.warmup()  # 异步预热所有股票的两种复权因子

# 3. 加载K线并复权（每只 <5ms）

for symbol in stock_pool:
    # 加载原始K线
    df_raw = load_kline(symbol, period='1d', ...)

    # 获取因子表（内存命中，<1ms）
    factor_table = service.get_factor_table(symbol, 'front')

    # 应用复权（<5ms）
    df_adjusted = service.apply_factors(df_raw, factor_table)

    # 开始回测计算...
```

### 3.2 性能指标

| 操作 | 时间 | 说明 |
|------|------|------|
| 预热 100 只股票 | 20-50 秒 | 一次性，异步执行 |
| 单只股票复权 | <5ms | 内存缓存命中 |
| 1000只股票 × 5年日线 | 5-10 秒 | 批量处理 |

### 3.3 注意事项

1. **只使用前复权**：回测通常只关心相对价格变化，前复权足够
2. **不需要后复权**：除非要精确匹配历史交易金额（极少需要）
3. **不需要持久化复权后数据**：实时应用足够快，避免存储冗余

## 5. 故障排查

### 5.1 常见问题

#### Q1: 首次加载还是很慢（>500ms）
- **检查**: 是否从网络获取 XDXR 数据（而非文件缓存）
- **解决**: 确保 `data/xdxr_cache/` 目录有写入权限，且已生成 .pkl 文件

#### Q2: 复权结果不对（价格异常）
- **检查**: XDXR 数据格式是否匹配（不同数据源字段不同）
- **解决**: 查看日志 `[FactorCalc]` 是否有警告，确认 category 字段存在

#### Q3: 内存占用过高
- **检查**: `TTLCache` 的 `maxsize` 设置
- **解决**: 调整 `AdjustmentFactorService._factor_cache` 的 maxsize（默认 500）

### 5.2 日志关键词

## 6. 附录：复权算法详解

### 6.1 前复权计算（通达信标准）✅

**适用范围**: 所有周期（日线、分钟线统一使用）

#### 算法原理

通达信前复权：让除权日之前的价格"下跌"到除权后的水平，消除价格跳空。

#### 计算公式

```python
# 1. 计算除权价（理论价）
除权价 = 收盘价 - 每股分红

# 2. 计算单日因子（< 1.0，让价格下跌）
单日因子 = 除权价 / 收盘价

# 3. 应用规则（关键）:
#    - 最新除权日及之后: 因子 = 1.0（价格不变）
#    - 最新除权日之前: 因子 = 最新除权日的单日因子（不累积）
```

#### 实际示例（茅台 600519.SH）

```
2025-12-19 除权除息：
┌─────────────────────────────────────────┐
│ 不复权收盘价:    1410.00 元              │
│ 每股分红:        23.957 元 (10派239.57)  │
│ 除权价（理论）:   1410.00 - 23.957     │
│                 = 1386.043 元            │
├─────────────────────────────────────────┤
│ 单日因子:        1386.043 / 1410.00     │
│                 = 0.983009               │
└─────────────────────────────────────────┘

价格调整：
┌─────────────────────────────────────────┐
│ 2025-12-18 (除权日前一天):              │
│   不复权: 1431.00 元                    │
│   前复权: 1431.00 × 0.983009 = 1407.04 元  │
│                                         │
│ 2025-12-19 (除权日当天):                │
│   不复权: 1410.00 元                    │
│   前复权: 1410.00 × 1.0 = 1410.00 元 ✓ │
│                                         │
│ 2025-12-22 (除权日后):                 │
│   不复权: 1408.26 元                    │
│   前复权: 1408.26 × 1.0 = 1408.26 元 ✓ │
└─────────────────────────────────────────┘
```

#### 关键点

1. **不累积**: 使用最新一次除权的因子，不是累积所有历史除权
2. **< 1.0 因子**: 让除权日前价格下跌（0.983009 < 1.0）
3. **除权日不变**: 除权日及之后价格保持原始值

#### 代码实现

**文件**: `backend/src/myquant/core/market/services/adjustment_factor_service.py`

```python
def _calculate_front_factors(self, symbol: str, xdxr_data: list) -> dict:
    """计算前复权因子表（通达信标准）"""

    # 1. 筛选除权记录
    dividend_records = xdxr_df[xdxr_df['category'] == 1].copy()

    # 2. 计算除权价
    dividend_per_share = fenhong / 10  # 每股分红
    ex_price = (close_prices - dividend_per_share) / (1 + songgu/10 + peigu/10)

    # 3. 计算单日因子（通达信标准：< 1.0）
    dividend_records['daily_factor'] = ex_price / close_prices

    # 4. 取最新除权日的因子（不累积）
    latest_daily_factor = dividend_records.iloc[0]['daily_factor']
    latest_ex_date = dividend_records['date'].max()

    # 5. 生成因子表
    daily_df['cumulative'] = daily_df['date'].apply(
        lambda d: 1.0 if d >= latest_ex_date else latest_daily_factor
    )

    return factor_dict
```

## 1. 数据源概述

### 1.1 支持的数据源

| 数据源 | 说明 | L0-L5支持 | 主要优势 | 主要限制 |
|--------|------|------------|----------|----------|
| **TdxQuant** | 通达信本地数据 | L1-L5完整 | ⚡L1最快、🎯L5板块、📊L4财务 | ⚠️K线受限、❌L0未验证、⚠️当天分钟不支持 |
| **XtQuant** | 迅投数据源 | L0、L3、L4部分 | 🚀L0订阅、📈L3 K线 | ❌L2历史、❌L5板块、❌L1快照较慢 |
| **PyTdx** | PyTdx库 | L1、L3部分 | 免费、开源 | ❌L2历史失败、⚠️性能较慢、❌L4失败 |
| **QLib** | 本地数据库 | 存储数据 | 📦历史数据完整、🔍性能优异 | 需要先下载或生成 |

### 1.2 数据源特点总结

**TdxQuant**：
- ✅ **L1实时快照**: 0.60ms最快（所有数据源中最快）
- ✅ **L5板块管理**: 板块数据是主要优势
- ✅ **L4财务数据**: GP1-GP46完整支持
- ✅ **L2历史快照**: 支持历史K线（XtQuant不支持）
- ⚠️ **L3 K线数据**: 最多1088根，不支持当天分钟
- ❌ **L0订阅推送**: 功能未验证，不建议使用

**XtQuant**：
- ✅ **L0订阅推送**: <1ms，真正的实时推送
- ✅ **L3 K线数据**: 完整支持，无数量限制
- ⚠️ **L1实时快照**: 0.90ms（比TdxQuant慢50%）
- ✅ **L2历史快照**: 支持历史K线（TdxQuant盘中不支持分钟K线）
- ❌ **L5板块管理**: 不支持
- ❌ **L4财务数据**: 不支持

**QLib本地数据库**：
- ✅ **历史数据完整**: 存储下载的所有数据
- ✅ **性能优异**: 本地读取，速度极快
- ✅ **稳定性高**: 不依赖网络连接
- ⚠️ **需要下载**: 需要先用其他数据源下载

## 3. 场景化选择策略

### 3.1 使用场景分类

#### 🎯 场景1：实时行情监控（L0 + L1）

**需求**：
- 订阅多只股票的实时行情
- 获取实时快照（价格、成交量等）
- 低延迟要求（<100ms）

**推荐方案**：**XtQuant订阅 + TdxQuant快照**

```
订阅: XtQuant (L0订阅，<1ms推送)
快照: TdxQuant (L1快照，0.60ms最快)
```

**原因**：
- XtQuant订阅延迟低，推送可靠
- TdxQuant快照最快，批量获取性价比高
- 两者结合：实时用XtQuant，快照用TdxQuant

**配置示例**：
```python
# 订阅关键股票（XtQuant）
subscription_source = XtQuantAdapter()
subscription_source.subscribe(important_stocks)

# 批量快照（TdxQuant）
snapshot_source = TdxQuantAdapter()
snapshot_source.get_batch_snapshots(all_stocks)
```

---

#### 📊 场景2：K线数据回测（L2/L3）

**需求**：
- 大量历史K线数据
- 长期数据（几个月到几年）
- 支持多周期（1m, 5m, 15m, 30m, 1d, 1w）
- 数据完整性和准确性要求高

**推荐方案**：**XtQuant下载 + QLib存储**

```
下载: XtQuant (无限制，完整支持)
存储: QLib (本地读取，极快)
读取: QLib (性能最佳)
```

**原因**：
- XtQuant K线无限制，适合批量下载
- QLib本地存储，读取性能极佳
- 避免每次回测都重新下载

**数据获取流程**：
```python
# 1. 使用XtQuant批量下载
xtquant_source = XtQuantAdapter()
xtquant_source.download_kline_batch(all_stocks, periods='1d', start='2020-01-01', end='2025-12-31')

# 2. 使用QLib本地读取
qlib = QLib()
kline_data = qlib.get_kline(stock_code, '1d', count=1000)

# 3. 结合实时数据更新
# 实时用XtQuant获取当天数据
# 历史用QLib读取
```

#### 🎯 场景3：板块分析（L5）

**需求**：
- 行业板块、概念板块数据
- 板块指数实时行情
- 板块成分股列表
- 板块成分股筛选

**推荐方案**：**TdxQuant（唯一选择）**

```
板块数据: TdxQuant (L5完整支持，是主要优势)
```

**原因**：
- TdxQuant是唯一完整支持板块管理的数据源
- 板块数据是TdxQuant的核心优势
- XtQuant和PyTdx都不支持板块数据

**使用示例**：
```python
# 获取板块列表
tdx_source = TdxQuantAdapter()
sector_list = tdx_source.get_sector_list()  # 返回所有板块

# 获取板块成分股
stocks_in_sector = tdx_source.get_stock_list_in_sector('白酒板块')

# 获取板块指数快照
sector_snapshot = tdx_source.get_market_snapshot('白酒板块指数')
```

- 大批量股票财务查询

**使用示例**：

### 5.2 智能路由策略

#### 自动路由：UnifiedDataManager

**MyQuant系统应该实现智能路由**：

```python
class UnifiedDataManager:
    """统一数据管理器"""

    def get_l0_subscription(self, stocks):
        """L0订阅推送 - 使用XtQuant"""
        return self.xtquant.subscribe(stocks)

        """L1实时快照 - 使用TdxQuant（最快）"""
        return self.tdxquant.get_batch_snapshots(stocks)

    def get_l2_history(self, stocks):
        """L2历史快照 - 使用TdxQuant"""
        return self.tdxquant.get_history_kline(stocks)

    def get_l2_today(self, stocks):
        """L2当天分钟 - 使用XtQuant"""
        return self.xtquant.get_today_minute(stocks)

    def get_l3_kline(self, stocks, count=1000):
        """L3 K线（大数量）- 从QLib读取"""
        return self.qlib.get_kline(stocks, count=count)

    def get_l4_financial(self, stocks):
        """L4财务数据 - 使用TdxQuant"""
        return self.tdxquant.get_financial_data(stocks)

    def get_l5_sector(self):
        """L5板块管理 - 使用TdxQuant"""
        return self.tdxquant.get_sector_data()
```

## 6. 故障切换与容灾

### 6.1 故障检测与切换

#### 自动故障检测

```python
class DataSourceHealthMonitor:
    """数据源健康监控"""

    def __init__(self):
        self.health_status = {
            'TdxQuant': 'unknown',
            'XtQuant': 'unknown',
            'QLib': 'unknown'
        }
        self.last_check_time = {}

    def check_tdxquant(self):
        """检查TdxQuant健康状态"""
        try:
            adapter = TdxQuantAdapter()
            start = time.time()
            snapshot = adapter.get_market_snapshot('600519.SH')
            elapsed = (time.time() - start) * 1000
            self.health_status['TdxQuant'] = 'healthy' if elapsed < 100 else 'slow'
            self.last_check_time['TdxQuant'] = time.time()
            return True
        except Exception as e:
            self.health_status['TdxQuant'] = 'unhealthy'
            return False

    def check_xtquant(self):
        """检查XtQuant健康状态"""
        try:
            # 检查订阅状态
            if self.xtquant.is_subscribed():
                self.health_status['XtQuant'] = 'healthy'
            else:
                self.health_status['XtQuant'] = 'warning'
            return True
        except Exception as e:
            self.health_status['XtQuant'] = 'unhealthy'
            return False

    def get_best_source_for_task(self, task_type):
        """根据任务类型和健康状态选择最佳数据源"""
        if task_type == 'L1_snapshot':
            if self.health_status.get('TdxQuant') == 'healthy':
                return 'TdxQuant'
            elif self.health_status.get('XtQuant') == 'healthy':
                return 'XtQuant'
            else:
                return 'QLib'
        # ... 其他任务类型
```

### 6.2 自动切换策略

```python
class DataSourceRouter:
    """数据源路由器 - 支持自动切换"""

    def __init__(self):
        self.current_source = {}
        self.failure_count = {}
        self.switch_threshold = 3  # 连续失败3次后切换

    def route_request(self, task_type, *args, **kwargs):
        """路由请求到最佳数据源"""
        sources = self.get_available_sources(task_type)

        # 尝试所有可用数据源
        for source in sources:
            try:
                result = self.call_source(source, task_type, *args, **kwargs)
                self.on_success(source, task_type)
                self.reset_failure_count(source)
                return result
            except Exception as e:
                self.on_failure(source, task_type, e)

        # 所有数据源都失败，使用备用方案
        return self.get_fallback_data(task_type, *args, **kwargs)

    def on_success(self, source, task_type):
        """请求成功时的处理"""
        self.current_source[task_type] = source
        self.log(f"{source} 成功: {task_type}")

    def on_failure(self, source, task_type, error):
        """请求失败时的处理"""
        count = self.failure_count.get(f"{source}_{task_type}", 0) + 1
        self.failure_count[f"{source}_{task_type}"] = count
        self.log(f"{source} 失败({count}/{self.switch_threshold}): {task_type} - {error}")

        if count >= self.switch_threshold:
            self.switch_source(task_type)

    def switch_source(self, task_type):
        """切换到备用数据源"""
        current = self.current_source.get(task_type)
        available = self.get_available_sources(task_type)
        if current in available:
            new_index = (available.index(current) + 1) % len(available)
            new_source = available[new_index]
            self.current_source[task_type] = new_source
            self.log(f"自动切换: {current} -> {new_source} (任务: {task_type})")
```

Level 2 (备用): 纯TdxQuant快照

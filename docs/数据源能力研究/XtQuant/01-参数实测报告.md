# XtQuant 参数实测报告

> 测试日期: 2026-02-05
> 测试工具: 直接调用 xtdata.get_market_data_ex
> 测试股票: 600519.SH (贵州茅台)

## 1. start_time/end_time 参数测试

### 测试1: 完全不传参数（推荐）

```python
result = xtdata.get_market_data_ex(
    field_list=['time', 'open', 'high', 'low', 'close', 'volume', 'amount'],
    stock_list=['600519.SH'],
    period='1d',
    count=16
    # 不传 start_time 和 end_time
)
```

**结果**: ✅ 成功
- 获取 16 条数据
- 最新日期: 20260204
- 最旧日期: 20260114

### 测试2: 传空字符串 ''

```python
start_time='',
end_time=''
```

**结果**: ✅ 成功
- 获取 16 条数据
- 与不传参数结果相同

### 测试3: 传 0

```python
start_time=0,
end_time=0
```

**结果**: ✅ 成功
- 获取 16 条数据
- 与不传参数结果相同

**结论**: 三种方式都有效，但**官方文档说"缺省"时用count，建议不传参数**


### 测试3: 字符串 'front' ✅

```python
dividend_type='front'
```

**结果**: ✅ 成功（前复权）

### 测试4: 字符串 'back' ✅

```python
dividend_type='back'
```

**结果**: ✅ 成功（后复权）

**结论**:
- ✅ **dividend_type 必须是字符串**: 'none', 'front', 'back'
- ❌ **不能是数字**: 0, 1, 2 会报错

**当前代码BUG**:
```python

# ✅ 正确：单个获取方法使用字符串
```

## 8. 关键发现总结

| 参数 | 有效值 | 无效值 | 建议 |
|------|--------|--------|------|
| start_time | 不传, '', 0 | - | **不传** |
| end_time | 不传, '', 0 | - | **不传** |
| dividend_type | 'none', 'front', 'back' (字符串) | 0, 1, 2 (数字) ❌ | **必须用字符串** |
| count | 正整数 | - | 根据需要设置 |
| period | '1d', '1m', '5m' 等 | - | 根据需要设置 |
| fill_data | True, False | - | True 填充缺失数据 |

## 9. 当前代码问题

### 问题1: dividend_type 类型错误

**位置**: `backend/data/adapters/xtquant_dual_instance_adapter.py`

**行号**: 842, 918

**错误代码**:
```python
# ❌ 批量获取方法
dividend_type_map = {"none": 0, "front": 1, "back": 2}
dividend_type = dividend_type_map.get(adjust_type, 1)  # 返回数字
```

**正确代码**:
```python
# ✅ 单个获取方法
dividend_type_map = {"none": 'none', "front": 'front', "back": 'back'}
dividend_type = dividend_type_map.get(adjust_type, 'front')  # 返回字符串
```

**影响**:
- 批量获取K线时会报错
- 单个获取K线正常

**修复优先级**: 🔴 **高优先级** - 导致批量功能完全不可用

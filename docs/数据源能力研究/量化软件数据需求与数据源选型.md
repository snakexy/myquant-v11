# 量化交易软件数据需求与数据源选型分析

> **创建日期**: 2026-03-04
> **分析重点**: 基于实际策略需求，从需求反推数据源架构设计
> **策略类型**: 技术分析、板块轮动、量价分析、T+0趋势

---

## 📋 分析框架

本文档从**量化交易软件的实际需求**出发，分析：
1. **盘中实时监控**需要哪些数据
2. **盘后数据处理**需要哪些数据
3. **每种数据类型**应该使用哪个数据源
4. **架构优化建议**

## 一、盘中实时监控数据需求

### 1.1 实时行情监控

#### 需求场景
- **技术分析**: 实时查看当前价格、涨跌幅、成交量
- **T+0趋势**: 监控趋势变化，及时判断买卖点
- **量价分析**: 实时监控量价关系变化

#### 数据类型需求

| 数据类型 | 数据内容 | 实时性要求 | 数据量 |
|---------|---------|-----------|--------|
| **实时价格** | 当前价、昨收、涨跌幅、涨跌额 | 高（秒级） | 单只或多只 |
| **实时成交量** | 成交量、成交额、换手率 | 高（秒级） | 单只或多只 |
| **买卖盘口** | 五档买卖价、五档成交量 | 高（秒级） | 单只 |
| **实时K线** | 分钟K线（开盘价、最高价、最低价、收盘价、成交量） | 中（分钟级） | 单只或多只 |

#### 数据源选型

```
┌─────────────────────────────────────────────────────────────┐
│ 实时价格、成交量                                       │
├─────────────────────────────────────────────────────────────┤
│ 策略A: 热点股票监控（<100只）                        │
│   → XtQuant订阅 (<1ms推送，L0缓存)                     │
│   优势: 真正推送，延迟最低                             │
│                                                       │
│ 策略B: 全市场扫描（5000+只）                          │
│   → TdxQuant快照 (0.60ms)                             │
│   优势: 速度最快，批量获取无频率限制                     │
│   示例: 获取全市场涨幅榜、成交量榜                      │
│                                                       │
│ 策略C: 单只股票深度监控                                 │
│   → XtQuant订阅 (优先) 或 TdxQuant快照 (备用)           │
└─────────────────────────────────────────────────────────────┘
```

**建议配置**：
- L0订阅缓存（热点股票50-100只）: XtQuant订阅
- L1实时快照（全市场）: TdxQuant（0.60ms最快）

### 1.2 热点股票监控

#### 需求场景
- **技术分析**: 识别当前热点股票，辅助决策
- **量价分析**: 分析热点股票的量价关系
- **T+0趋势**: 捕捉趋势启动的信号

#### 数据类型需求

| 数据类型 | 数据内容 | 更新频率 | 扫描范围 |
|---------|---------|---------|---------|
| **涨幅榜** | 按涨幅排序的股票列表 | 30秒 | 全市场5000+只 |
| **成交量榜** | 按成交量排序的股票列表 | 30秒 | 全市场5000+只 |
| **涨速榜** | 按涨速排序的股票列表 | 30秒 | 全市场5000+只 |
| **大单监控** | 大单成交记录 | 实时 | 热点股票 |

#### 数据源选型

```
┌─────────────────────────────────────────────────────────────┐
│ 涨幅榜、成交量榜、涨速榜（全市场扫描）                 │
├─────────────────────────────────────────────────────────────┤
│ 数据源: TdxQuant (0.60ms)                             │
│ 优势:                                                 │
│   • 速度最快（0.60ms，比XtQuant 0.90ms快33%）        │
│   • 无频率限制（1000次并发100%成功）                    │
│   • 批量获取优势（10股批量快1.66x）                     │
│   • 24/7可用（不受交易时间限制）                         │
│                                                       │
│ 实现方式:                                              │
│   1. 每30秒调用 TdxQuant.get_market_data() 获取全市场数据 │
│      (直接获取：当前价、昨收、成交量、成交额等)              │
│   2. 计算涨跌幅和涨速等衍生指标                         │
│      (涨跌幅 = (当前价 - 昨收) / 昨收 × 100%)              │
│   3. 按涨跌幅、成交量排序生成榜单                        │
│   4. 缓存到Redis（TTL 30秒）                          │
└─────────────────────────────────────────────────────────────┘
```

**建议配置**：
- 数据源: **TdxQuant**（首选，0.60ms）
- 刷新频率: 30秒
- 缓存策略: Redis L1缓存（TTL 30秒）

#### 需求场景

#### 数据类型需求

| 数据类型 | 数据内容 | 实时性要求 | 数据量 |

| **大单成交** | 大单成交记录（价格、成交量、时间） | 实时 | 单只或多只 |
| **委买卖比** | 委买总量、委卖总量 | 高（秒级） | 单只 |

#### 数据源选型

```
┌─────────────────────────────────────────────────────────────┐
│ 买卖盘数据（单只股票深度监控）                          │
├─────────────────────────────────────────────────────────────┤
│ 数据源: TdxQuant (0.60ms)                             │
│ 优势:                                                 │
│   • 支持买卖盘数据（GP字段）                           │
│   • 盘中辅助信息丰富                                   │
│   • 速度最快（0.60ms）                                 │
│                                                       │
│ 适用场景:                                              │
│   • 监控重点股票的盘口变化                             │

│   • 判断资金流向                                       │
│                                                       │
│ 实现方式:                                              │
│   1. 每1-3秒调用 TdxQuant.get_market_data() 获取盘口数据  │
│   2. 缓存到Redis（TTL 5秒）                           │
│   3. 实时更新前端显示                                 │
└─────────────────────────────────────────────────────────────┘
```

**建议配置**：
- 数据源: **TdxQuant**（首选，0.60ms）
- 刷新频率: 1-3秒
- 缓存策略: Redis L1缓存（TTL 5秒）

### 1.4 实时K线更新

#### 需求场景
- **技术分析**: 实时查看分钟K线，分析趋势
- **T+0趋势**: 实时K线用于判断趋势变化
- **量价分析**: 分钟K线用于分析量价关系

#### 数据类型需求

| 数据类型 | 数据内容 | 更新频率 | 数据量 |
|---------|---------|---------|--------|
| **分钟K线** | 开盘价、最高价、最低价、收盘价、成交量 | 每分钟 | 单只或多只 |
| **分时K线** | 分时价格、成交量 | 每分钟 | 单只或多只 |

#### ⚠️ 关键问题

**TdxQuant不支持当天分钟数据！**

从源代码和测试确认：
- ✅ TdxQuant可以获取历史分钟数据
- ❌ TdxQuant**不支持当天分钟数据**
- ✅ XtQuant可以获取当天分钟数据

#### 数据源选型

```
┌─────────────────────────────────────────────────────────────┐
│ 当天分钟K线（实时更新）                               │
├─────────────────────────────────────────────────────────────┤
│ 数据源: XtQuant (0.90ms)                             │
│ 原因:                                                 │
│   • TdxQuant不支持当天分钟数据 ❌                        │
│   • XtQuant支持当天分钟数据 ✅                           │
│   • 交易时间可用 ✅                                   │
│                                                       │
│ 实现方式:                                              │
│   1. 每1分钟调用 XtQuant.get_market_kline() 获取分钟K线  │
│   2. 缓存到Redis（TTL 60秒）                          │
│   3. 实时更新前端显示                                 │
│   4. 盘后归档到QLib数据库                              │
│                                                       │
│ 缓存策略:                                              │
│   • 盘中: 实时获取（XtQuant 0.90ms）                   │
│   • 盘后: 从QLib读取（7-10ms）                         │
└─────────────────────────────────────────────────────────────┘
```

**建议配置**：
- 盘中: **XtQuant**（0.90ms，支持当天分钟数据）
- 盘后: **QLib数据库**（7-10ms）
- 缓存策略: Redis L2缓存（TTL 60秒）

### 1.5 板块轮动监控

#### 需求场景
- **板块轮动**: 识别热点板块，捕捉轮动机会
- **技术分析**: 分析板块指数走势
- **量价分析**: 分析板块成分股的整体表现

#### 数据类型需求

| 数据类型 | 数据内容 | 更新频率 | 数据量 |
|---------|---------|---------|--------|
| **板块列表** | 所有板块名称、代码 | 低（每日） | 586个 |
| **板块成分股** | 每个板块的成分股列表 | 低（每日） | 5000+只 |
| **板块涨幅** | 板块指数涨跌幅 | 高（分钟级） | 586个 |
| **板块成分股表现** | 板块内股票的涨跌幅排序 | 高（分钟级） | 板块内股票 |

#### 数据源选型

```
┌─────────────────────────────────────────────────────────────┐
│ 板块数据（TdxQuant绝对优势）                            │
├─────────────────────────────────────────────────────────────┤
│ 数据源: TdxQuant (6.99ms, 586板块)                  │
│ 优势:                                                 │
│   • 速度极快（586板块仅需6.99ms）                      │
│   • 数据完整（板块指数、成分股）                        │
│   • 板块相关数据的绝对优势                               │
│   • 24/7可用                                          │
│                                                       │
│ 实现方式:                                              │
│   1. 调用 TdxQuant.get_sector_list() 获取板块列表        │
│      (6.99ms, 586个板块)                              │
│   2. 调用 TdxQuant.get_stock_list_in_sector() 获取成分股  │
│      (测试: 银行板块, 202个股票, <10ms)                │
│   3. 调用 TdxQuant.get_market_data() 获取板块指数涨跌和成分股数据   │
│      (0.60ms)                                         │
│   4. 计算成分股涨跌幅（涨跌幅 = (当前价 - 昨收) / 昨收 × 100%）      │
│   5. 排序生成热点板块榜                                │
│   6. 缓存到Redis（板块列表TTL 1天，涨幅TTL 30秒）      │
└─────────────────────────────────────────────────────────────┘
```

**建议配置**：
- 数据源: **TdxQuant**（首选，绝对优势）
- 板块列表更新: 每日
- 板块涨幅刷新: 30秒
- 缓存策略: Redis（板块列表TTL 1天，涨幅TTL 30秒）

| **分钟K线** | 1分钟、5分钟K线 | 每日 | 5000+只 |
| **复权数据** | 前复权、后复权因子 | 每日 | 5000+只 |

#### 数据源选型

```
┌─────────────────────────────────────────────────────────────┐
│ K线历史数据下载（盘后批量）                             │
├─────────────────────────────────────────────────────────────┤
│ 数据源: TdxQuant (24/7可用, 批量1.66x优势)            │
│ 优势:                                                 │
│   • 24/7可用（盘后可用，不受交易时间限制）              │

└─────────────────────────────────────────────────────────────┘
```

- 批量大小: 50-100股/批次
- 数据归档: QLib数据库

### 2.3 因子计算

| 数据类型 | 数据内容 | 计算频率 | 数据来源 |

| **量价因子** | 量价关系、成交活跃度等 | 盘后/实时 | K线+成交数据 |
| **趋势因子** | 趋势强度、方向等 | 盘后/实时 | K线数据 |

#### 数据源选型

```
┌─────────────────────────────────────────────────────────────┐
│ 因子计算（依赖QLib历史数据）                           │
├─────────────────────────────────────────────────────────────┤
│ 数据源: QLib数据库                                    │
│ 优势:                                                 │
│   • 数据已格式化，计算方便                             │
│   • 速度较快（7-10ms）                               │
│   • 支持复权数据                                     │
│   • 历史数据完整                                     │
│                                                       │
│ 实现方式:                                              │
│   1. 从QLib读取K线历史数据                             │
│   2. 计算技术指标（MACD、KDJ等）                      │
│   3. 计算量价因子                                     │
│   4. 计算趋势因子                                     │
│   5. 缓存因子结果到Redis（TTL 1小时）                │
│                                                       │
│ 缓存策略:                                              │
│   • 因子结果: Redis（TTL 1小时）                        │
│   • 原始K线: QLib数据库                                │
└─────────────────────────────────────────────────────────────┘
```

**建议配置**：
- 数据源: **QLib数据库**（首选）
- 计算时间: 盘后计算历史因子，实时计算实时因子
- 缓存策略: Redis（TTL 1小时）

## 三、数据类型-数据源选型汇总表

### 3.1 盘中实时监控

## 五、核心结论

### 5.1 盘中实时监控

**核心数据源分工**：
- **TdxQuant**（主要）: 全市场扫描、板块数据、买卖盘监控
- **XtQuant**（补充）: 热点股票订阅、当天分钟K线

**关键优化点**：
1. **全市场扫描**: TdxQuant（0.60ms）比XtQuant（0.90ms）快33%
2. **板块数据**: TdxQuant有绝对优势（586板块6.99ms）
3. **当天分钟K线**: 明确使用XtQuant（TdxQuant不支持）

### 5.2 盘后数据处理

**核心数据源分工**：
- **TdxQuant**: K线历史数据下载（24/7，批量1.66x）
- **XtQuant**: 财务数据更新（首选）
- **QLib数据库**: 因子计算、策略回测（已格式化）

**关键优化点**：
1. **K线下载**: TdxQuant（24/7可用）可随时下载，不受交易时间限制
2. **批量下载**: 50-100股/批次，1.49x优势
3. **数据归档**: 统一归档到QLib，提高复用性

### 5.3 整体架构

**数据源优先级总结**：

```
盘中实时监控:
  L0 热点股票订阅 → XtQuant订阅 (<1ms)
  L1 全市场快照 → TdxQuant (0.60ms)
  L2 当天分钟K线 → XtQuant (0.90ms)

盘后数据处理:
  K线历史下载 → TdxQuant (24/7, 批量1.66x)
  财务数据更新 → XtQuant (首选) / TdxQuant (备用)
  因子计算 → QLib数据库 (已格式化)
  策略回测 → QLib数据库 (历史数据完整)
```
